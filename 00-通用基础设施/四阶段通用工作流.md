# 四阶段通用工作流（母版）

> 适用范围：一审程序、二审程序、执行异议之诉等所有具备"起诉-审理-判决"完整诉讼结构的程序。
> 非诉程序（执行实施、执行审查等）不适用本工作流，应使用专门的线性/审查工作流。

---

## 阶段一：材料分析与策略制定

### 输入
- 原始案件材料（根据程序类型不同，材料范围不同）
- 关联案件材料（如有在先诉讼/执行/仲裁案件）
- 法院来文（如有：受理通知、举证通知、传票等）

### 核心任务
1. **材料收集与编目**
   - 列出已识别材料清单，分类统计
   - 评估完整性，指出缺失材料
   - 生成《待补充材料清单》（含优先级、获取方式、负责方、期限）
   - 原始材料仅复制整理，不移动、不修改、不删除原件

2. **案由识别与适配**
   - 判断本案是否为普通案由（合同纠纷、侵权纠纷等）
   - 如为特殊案由（代位权、撤销权、人格否认、股东出资、执行异议之诉等）：
     - 调用 `案由适配索引.md` 中的对应指引
     - 按该案由的特殊要求调整材料范围和策略方向

3. **关联案件索引**
   - 如与在先案件有关联（如人格否认建立在原债权判决基础上）：
     - 列出关联案件清单（案号、法院、裁判结果、当前状态）
     - 在材料索引中标注关联关系
     - 评估关联案件裁判结果对本案的约束或支撑作用

4. **事实梳理**
   - 按时间顺序提取关键事件，生成《案件事实时间轴》
   - 每个事件标注：日期、内容、相关证据、性质（有利/不利/待核实）

5. **法律检索**
   - 检索相关法条、司法解释、指导性案例
   - 生成《法条汇编》
   - ⚠️ AI 检索能力有限，需提示用户在专业数据库（北大法宝、MetaLaw 等）中深度补充
   - 所有 AI 检索案例必须标明来源 URL，严禁编造

6. **策略制定**
   - 基于材料、事实、法律，生成《诉讼/应诉策略方案书》
   - 明确：代理方向（原告/被告）、核心主张、请求权/抗辩权基础、风险提示、Plan B

### 输出
- 《材料索引与分类.md》
- 《待补充材料清单.md》
- 《案件事实时间轴.md》
- 《法条汇编.md》
- 《诉讼/应诉策略方案书.md》

## 阶段二：文书制备与来文管理

### 输入
- 阶段一全部产出
- 客户情况说明（如有）
- 对方来文（如有：起诉状、答辩状、证据材料等）
- 法院来文（如有：受理通知、举证通知、传票等）

### 核心任务
1. **角色判定**
   - 根据策略方案书和客户情况说明，确认本方角色
   - 如角色不明或矛盾，询问用户确认
   - 调用 `角色判定规则.md` 确认诉讼地位

2. **期限提取与跟踪**
   - 读取全部法院来文，提取指定期限：
     - 举证期限（举证通知书）
     - 开庭日期（传票）
     - 答辩期限（应诉通知书）
   - 如法院未指定，适用法定期限
   - 生成《关键期限跟踪表》，附于每份产出文件

3. **起诉状/答辩状制备**
   - 首部：当事人信息完整，与材料索引一致
   - 案由：与策略方案书一致
   - 诉讼请求/抗辩意见：精确、具体、金额有计算依据
   - 事实与理由：按时间轴顺序客观陈述，不穿插证据编号
   - 尾部：具状人、日期、附注齐全

4. **证据目录与证据说明**
   - 证据目录：4 列表格（序号/证据名称/证明目的/页码），编号规则统一
   - **排除规则**：案件背景说明、律师工作笔记、待补充材料清单等内部材料不纳入证据目录
   - 证据说明（双版本）：
     - 自用版：含证据形式、来源、内容摘要、证明力评估（强/中/弱）、预判质证
     - 提交法院版：段落式排版，客观陈述证据事实，关键处用 `>` 引用格式强调

5. **辅助文书（按需）**
   - 财产保全申请书
   - 调查取证申请书
   - 诉讼费用计算表
   - 管辖权异议申请书（如适用）

6. **来文管理**
   - 对方来文放入 `对方来文/` 文件夹
   - 法院来文放入 `法院来文/` 文件夹
   - 建立来文接收记录

### 输出
- 《起诉状.md》/《答辩状.md》
- 《证据目录.md》
- 《证据说明（自用版）.md》
- 《证据说明（提交法院版）.md》
- 辅助文书（按需）
- 《关键期限跟踪表.md》

## 阶段三：庭审/听证准备

### 输入
- 阶段二全部产出
- 对方新提交的材料（如有）
- 法院送达的对方材料（如有）

### 核心任务
1. **质证意见**
   - 对对方每份证据按"四性"分析：真实性、合法性、关联性、证明力
   - 增加策略性评价：对我方有利证据主动"借力打力"，转化为己方论据
   - 预判对方可能提出的质疑及应对口径

2. **开庭提纲（内部文件，不提交法院）**
   - 程序事项核对清单
   - 法官可能发问的预判及回答口径
   - 回答禁区：明确哪些话不能说
   - 我方发问设计
   - 争议焦点梳理
   - 辩论要点（第一轮/第二轮/最后陈述）
   - 突发情况应对预案

3. **庭审纪要模板（庭前准备空模板）**
   - 基本信息、程序事项、法庭调查摘要、法庭辩论摘要、其他安排
   - 采用稀疏排版，打印后方便手写补充

### 输出
- 《质证意见.md》（需提交法院）
- 《开庭提纲.md》（内部使用）
- 《庭审纪要.md》（庭后填写）

## 阶段四：收官与程序转换

### 输入
- 阶段三全部产出
- 庭审后补充材料（如有）
- 法院裁判文书（判决书/裁定书/调解书）
- 法院指定期限（上诉期、履行期等）

### 核心任务
1. **庭后代理词**
   - 基于庭审纪要，系统梳理已查明事实和法律
   - 回应庭审中法官归纳的争议焦点
   - 不提出新的事实主张（除非有正当理由且对方无异议）
   - 在判决作出前提交

2. **裁判应对**
   - 读取判决书/裁定书，确认结果、上诉期限、履行期限、诉讼费承担
   - 根据结果分流：
     - 败诉/部分败诉 + 当事人决定上诉 → 准备上诉状
     - 胜诉 + 对方不履行 → 准备强制执行申请
     - 调解结案 → 确认调解书效力及履行安排

3. **期限管理与程序转换**
   - 调用 `程序转换索引.md`，确认下一步程序路径
   - 生成《关键期限跟踪表》更新版
   - 关键期限到达前 3 日提醒用户

4. **退费与归档（如适用）**
   - 判决生效后，确认诉讼费退费条件
   - 生成退费申请材料
   - 案件终局后生成《结案报告》+《复盘总结》

### 输出
- 《代理词.md》
- 《上诉状.md》/《强制执行申请书.md》
- 《退费申请书.md》（如适用）
- 《结案报告.md》+《复盘总结.md》（案件终局后）
- 更新后的《关键期限跟踪表.md》

---

## 通用原则（贯穿四阶段）

1. **原始材料保护**：AI 不移动、不修改、不删除原始文件
2. **双版本设计**：关键文书区分"内部工作版"与"提交法院版"
3. **AI 边界声明**：所有生成文书标注"初稿，需律师审核后使用"
4. **严禁编造**：AI 检索案例必须标明来源 URL
5. **期限优先**：法院指定 > 法定期限，关键期限到达前主动提醒

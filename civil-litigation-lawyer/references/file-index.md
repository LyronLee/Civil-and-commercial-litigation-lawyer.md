## content.md 结构速查

```
一、通用基础设施（跨程序共享）
   1.1 四阶段通用工作流母版（阶段一~阶段四 + 通用原则）
   1.2 案由适配索引（5种特殊案由：代位权/撤销权/人格否认/股东出资/执行异议之诉）
   1.3 程序转换索引（一审→二审/执行/再审；二审→再审/执行/发回；再审→检察监督/执行回转/发回；执行→各模块）
   1.4 角色判定规则（原告/被告/第三人/案外人/申请执行人/被执行人）
   1.5 质量检查清单（通用检查 + 四阶段检查 + 特殊案由检查）

二、一审程序（四阶段骨架）
   阶段一：战略把脉（材料收集→案由识别→事实梳理→法律检索→策略制定）
   阶段二：诉讼材料及来文管理（角色判定→期限提取→起诉状/答辩状→证据目录→辅助文书→来文管理）
   阶段三：庭审交锋（质证意见→开庭提纲→庭审纪要）
   阶段四：收官与执行（代理词→裁判应对→期限管理→退费归档→结案复盘）

三、二审程序（四阶段骨架）
   核心特点、触发条件、前置依赖、关键期限
   阶段一：战略分析（判决书解析→错误诊断→改判评估→上诉策略）
   阶段二：材料制备（上诉状/二审答辩状→新证据目录→辅助文书）
   阶段三：庭审准备（二审质证→开庭提纲→庭审纪要）
   阶段四：收官（代理词→裁判应对→再审申请/执行申请→结案复盘）

四、再审程序（三阶段）
   法条更新说明（2023年修正《民事诉讼法》新旧条号对照）
   关键期限（6个月除斥期间/特殊4项/检察监督2年）
   阶段一：再审申请（原审裁判分析→再审事由匹配→成功率预测→新证据评估→申请书起草→询问/听证准备）
   阶段二：再审审理（按原审程序重新审理，配备独立四阶段工作流）
   阶段三：收官（裁判结果分析→后续路径选择→退费归档→结案复盘）

五、执行程序（四大模块）
   总入口：触发条件、前置检查、四大模块路由、关键期限对照表、新旧条号对照
   模块一：执行实施主线（申请执行→财产查控→信用惩戒→悬赏执行→执行和解）
   模块二：执行审查程序（行为异议→案外人异议→追加异议→复议→执行监督）
   模块三：执行异议之诉（4个子类型：案外人/申请执行人/追加变更/分配方案）
   模块四：执行衍生与特殊救济（执行回转→参与分配→执行协调→执行救助）

六、跨程序通用规则
   期限计算通用规则、送达规则、新旧法条衔接、法条引用规范
```

## templates.md 结构速查

```
一、一审程序模板
   阶段一：材料索引与分类、案件事实时间轴、类案检索报告、法条汇编、诉讼策略方案书
   阶段二：起诉状、答辩状、证据目录、证据说明（双版本）、财产保全申请、调查取证申请、诉讼费用计算表、来文管理记录
   阶段三：质证意见、开庭提纲、庭审纪要模板
   阶段四：代理词、上诉状/强制执行申请书、退费申请书、结案报告、复盘总结

二、二审程序模板
   阶段一：判决书分析意见、上诉策略方案书
   阶段二：上诉状、二审答辩状、新证据目录、证据说明
   阶段三：二审质证意见、二审开庭提纲、二审庭审纪要
   阶段四：二审代理词、裁判应对方案、再审申请书、强制执行申请书、结案复盘

三、再审程序模板
   阶段一：再审申请书、类案检索报告、原审裁判分析意见、关键期限跟踪表、询问/听证准备提纲
   阶段二：再审开庭提纲、再审庭审纪要、再审答辩状（被申请人视角）
   阶段三：再审裁判应对方案、检察监督申请书、结案报告、复盘总结

四、执行程序模板
   模块一：强制执行申请书、财产查控记录、限制消费/失信申请书、悬赏执行申请、执行和解协议
   模块二：执行异议申请书、案外人异议申请书、追加被执行人申请书、复议申请书、执行监督申请书
   模块三：四种执行异议之诉起诉状（案外人/申请执行人/追加变更/分配方案）
   模块四：执行回转申请、参与分配申请、执行协调申请、执行救助申请

五、通用模板规则
   文书命名规范、双版本设计规范、AI标注规范、法院格式要求
```

# 质量检查清单（Quality Check）

> 本文件包含各阶段产出的质量检查标准。
> 每个阶段结束时自动执行，不通过则修正。
> 适用程序：一审、二审、再审、执行程序、执行异议之诉、执行审查程序、执行衍生与特殊救济。

---

## 一、通用质量检查（所有产出）

### 1.1 格式检查

| 序号 | 检查项 | 合格标准 | 检查方法 |
|------|--------|---------|---------|
| 1.1.1 | 文件编码 | UTF-8 | 确认无乱码 |
| 1.1.2 | 标题层级 | 使用Markdown标准标题（# ## ###） | 目视检查 |
| 1.1.3 | 日期格式 | YYYY年MM月DD日 | 正则检查 |
| 1.1.4 | 金额格式 | 阿拉伯数字+中文大写 | 目视检查 |
| 1.1.5 | 法条引用 | 《法律名称》第X条第X款，标注"经[数据库名称]核实，现行有效" | 正则检查 |
| 1.1.6 | AI产出标注 | 文件开头标注"初稿，需律师审核后使用" | 目视检查 |
| 1.1.7 | 内部文件标注 | 开庭提纲、庭审纪要、策略方案书等标注"内部使用，不提交法院" | 目视检查 |
| 1.1.8 | 当事人信息完整 | 姓名/名称、住所、联系方式、法定代表人（如适用）完整 | 目视检查 |

### 1.2 内容完整性检查

| 序号 | 检查项 | 合格标准 |
|------|--------|---------|
| 1.2.1 | 当事人信息 | 姓名/名称、住所、联系方式、法定代表人（如适用）完整 |
| 1.2.2 | 诉讼请求 | 明确、具体、可执行、无歧义 |
| 1.2.3 | 事实陈述 | 客观、按时间顺序、有证据支撑 |
| 1.2.4 | 理由阐述 | 法律依据充分、逻辑清晰 |
| 1.2.5 | 证据清单 | 编号连续、名称准确、证明目的具体 |
| 1.2.6 | 管辖依据 | 明确引用管辖连接点 |
| 1.2.7 | 尾部要素 | 具状人/答辩人、日期、附注（副本份数、证据份数）齐全 |
| 1.2.8 | 格式符合法院要求 | 如需提交Word版本，已按受理法院要求调整字体（通常宋体小四/四号）和格式 |

### 1.3 策略性检查

| 序号 | 检查项 | 合格标准 |
|------|--------|---------|
| 1.3.1 | 诉讼时效 | 未届满或已说明中断/中止事由 |
| 1.3.2 | 管辖正确 | 级别管辖+地域管辖均正确 |
| 1.3.3 | 当事人适格 | 原告与本案有直接利害关系 |
| 1.3.4 | 诉的利益 | 诉讼请求具有现实争议性 |
| 1.3.5 | 证据链完整 | 关键事实均有证据支撑 |
| 1.3.6 | 风险披露 | 已提示主要法律风险 |
| 1.3.7 | 期限跟踪 | 已建立《关键期限跟踪表》，关键期限到达前3日提醒用户 |
| 1.3.8 | 法院指定期限提取 | 已读取全部法院来文，提取指定举证期限、开庭日期、答辩期限等 |
| 1.3.9 | 期限冲突处理 | 如法院指定与法定期限不一致，以法院指定为准，并在文书中注明来源和依据 |

### 1.4 保密与风险提示

| 序号 | 检查项 | 合格标准 |
|------|--------|---------|
| 1.4.1 | AI边界声明 | 所有AI生成文书已注明"初稿，需律师审核后使用" |
| 1.4.2 | 内部文件标注 | 开庭提纲、庭审纪要、策略方案书等内部文件已标注"内部使用，不提交法院" |
| 1.4.3 | 敏感信息脱敏 | 如涉及隐私上传AI，身份证号、银行账户、具体金额等已脱敏或用代号代替 |
| 1.4.4 | 期限风险告知 | 关键期限（上诉期、举证期、异议期）已明确告知客户，并留存告知记录 |
| 1.4.5 | 禁止编造声明 | AI检索到的案例已标明来源URL，未编造案例；法条引用已标注"经[数据库名称]核实" |
| 1.4.6 | 原始材料保护 | 原始案件材料仅复制整理，未移动、未修改、未删除原件 |
| 1.4.7 | 内部材料隔离 | 案件背景说明、律师工作笔记、待补充材料清单等内部文件已放入"内部参考材料"文件夹，与诉讼证据物理隔离 |

## 二、阶段一质量检查（材料分析与策略制定）

### 2.1 材料索引与分类

| 序号 | 检查项 | 合格标准 |
|------|--------|---------|
| 2.1.1 | 材料清单完整 | 所有已提供材料均已列入 |
| 2.1.2 | 分类合理 | 按逻辑类别分类，不少于3类 |
| 2.1.3 | 来源标注 | 每份材料标注来源和可靠性 |
| 2.1.4 | 缺失材料清单 | 独立文件，明确列出缺失材料、获取方法、优先级、负责方、截止日期 |
| 2.1.5 | 材料完整性评级 | 已评级：A级（材料齐全）/ B级（基本够用）/ C级（严重不足） |
| 2.1.6 | 关联案件索引 | 如与在先案件有关联，已建立关联案件清单并标注影响关系 |

### 2.2 案件事实时间轴

| 序号 | 检查项 | 合格标准 |
|------|--------|---------|
| 2.2.1 | 时间覆盖 | 覆盖所有关键日期 |
| 2.2.2 | 证据对应 | 每个事实有证据支撑 |
| 2.2.3 | 性质标注 | 区分有利/不利/待核实/中性 |
| 2.2.4 | 争议焦点 | 预判不少于2个争议焦点 |
| 2.2.5 | 日期精确度 | 尽量精确到具体日期，如无法确定则标注时间段 |
| 2.2.6 | 客观陈述 | 事件内容客观陈述，避免主观评价 |

### 2.3 类案检索报告

| 序号 | 检查项 | 合格标准 |
|------|--------|---------|
| 2.3.1 | 检索执行记录 | 必须记录检索平台、关键词、检索时间、结果数量 |
| 2.3.2 | 检索目标 | 明确不少于2个争议点 |
| 2.3.3 | 案例来源 | 每个案例必须有可核实的来源URL，严禁编造案号 |
| 2.3.4 | 无案例处理 | 如未检索到案例，必须如实说明，不得虚构填充 |
| 2.3.5 | 案例信息完整 | 每个案例标注案号、法院、裁判要旨、来源URL、检索状态 |
| 2.3.6 | 对比分析 | 有案例对比分析（如检索到案例） |
| 2.3.7 | 无编造声明 | 报告头部或尾部必须有"严禁编造案例"的声明 |
| 2.3.8 | 检索工具使用 | 必须证明使用了法律数据库（华宇元典法律数据库 yuandian_law）或专业数据库进行实际检索 |
| 2.3.9 | 专业数据库优先 | 优先检索人民法院案例库、裁判文书网、北大法宝、法信等专业数据库 |
| 2.3.10 | 不利案例检索 | 必须检索不利案例，只找有利案例会导致策略失误 |
| 2.3.9 | 不利案例检索 | 必须检索不利案例，只找有利案例会导致策略失误 |
| 2.3.10 | 案例时效性 | 优先检索近三年案例 |

**类案检索质量检查红线**：
- 如发现报告中存在无法核实来源的案例 → **不通过**，必须删除或标注"待核实"
- 如报告中案例无来源URL或来源URL为虚假链接 → **不通过**
- 如经检查未执行实际检索（无检索执行记录） → **不通过**
- 如未检索到案例但报告中有案例内容 → **不通过**（涉嫌编造）

### 2.4 法条汇编

| 序号 | 检查项 | 合格标准 |
|------|--------|---------|
| 2.4.1 | 分类整理 | 实体法、司法解释、程序法分类 |
| 2.4.2 | 适用要点 | 每个法条标注适用要点 |
| 2.4.3 | 现行有效 | 引用现行有效法律 |
| 2.4.4 | 条文准确 | 条文内容准确 |
| 2.4.5 | 层级检索 | 先法律→行政法规→司法解释→地方性规定 |
| 2.4.6 | 新旧对比 | 如法律有修订，对比新旧条文差异 |
| 2.4.7 | 数据库核实 | 每条法条标注"经[数据库名称]核实，现行有效" |
| 2.4.8 | 核实记录 | 必须记录核实平台、核实时间、核实结果 |
| 2.4.9 | 失效法条处理 | 如核实发现法条已修订/废止，引用现行有效版本并标注修订情况 |
| 2.4.10 | 无法核实标注 | 如无法核实某法条，标注"未经数据库核实，需人工确认" |

### 2.5 诉讼策略方案书

| 序号 | 检查项 | 合格标准 |
|------|--------|---------|
| 2.5.1 | 基本情况 | 案件基本情况完整 |
| 2.5.2 | 事实分析 | 有利/不利/待补强事实分析 |
| 2.5.3 | 法律分析 | 请求权基础或抗辩权基础明确 |
| 2.5.4 | 策略完整 | 核心策略+具体战术+Plan B |
| 2.5.5 | 风险提示 | 不少于2个风险提示及应对措施 |
| 2.5.6 | 工作计划 | 有工作计划与时间表 |
| 2.5.7 | 案由识别 | 已确认案由，如为特殊案由已调用案由适配索引 |
| 2.5.8 | 诉讼时效核查 | 如为被告，已核查诉讼时效并确定抗辩策略（如适用） |

### 2.6 特殊案由阶段一专属检查项

| 案由 | 检查项 |
|------|--------|
| 债权人代位权 | [ ] 已书面催告债务人限期向次债务人主张权利，并保留催告凭证；[ ] 次债务人的民事主体资格已确认；[ ] 代位行使的债权范围未超过债权人对债务人的到期债权额；[ ] 债务人未进入破产程序 |

| 执行异议之诉 | [ ] 前置执行异议裁定已收到并已读取；[ ] 起诉期限（15日）已起算并跟踪；[ ] 执行标的的权属证据材料完整；[ ] 权利形成时间早于查封时间的证据已收集 |

## 四、阶段三质量检查（庭审准备与交锋）

### 4.1 质证意见

| 序号 | 检查项 | 合格标准 |
|------|--------|---------|
| 4.1.1 | 逐份质证 | 对对方每份证据逐一质证，无遗漏 |
| 4.1.2 | 四性质证 | 真实性、合法性、关联性、证明目的 |
| 4.1.3 | 理由具体 | 不认可时说明具体理由 |
| 4.1.4 | 总结完整 | 有质证总结（对方证据体系缺陷、对我方有利事实总结、我方反驳口径） |
| 4.1.5 | 策略性评价 | 对我方有利证据已标注"借力打力"的转化利用方案 |
| 4.1.6 | 预先设防 | 预判对方可能提出的质疑及应对口径已准备 |
| 4.1.7 | 借力打力 | 对方用来证明A的证据若同时证明B（我方主张），已将其"反转为"我方论据 |

### 4.2 开庭提纲

| 序号 | 检查项 | 合格标准 |
|------|--------|---------|
| 4.2.1 | 流程预判 | 预判法庭调查、举证质证、法庭辩论阶段 |
| 4.2.2 | 发问应对 | 设计法官可能发问及应对（发问角度/回答口径/回答禁区/延伸应对） |
| 4.2.3 | 应急预案 | 制定对方突发观点的应对（对方当庭提交新证据/变更诉讼请求/法官倾向对方/调解方案/证人不利证言） |
| 4.2.4 | 程序核对 | 含当事人身份核对、代理人授权范围、回避申请预判、管辖权异议状态 |
| 4.2.5 | 我方发问 | 设计向对方发问的问题清单（目的/预期回答/对方回避时的追问/风险提示） |
| 4.2.6 | 争议焦点 | 梳理不少于2个争议焦点（我方立场/核心支撑证据/对方可能的抗辩/我方反驳要点） |
| 4.2.7 | 辩论要点 | 第一轮辩论/第二轮辩论补充/最后陈述（建议控制在3分钟内） |
| 4.2.8 | 调解预案 | 调解底线/最优调解方案/调解话术准备/当事人授权确认 |
| 4.2.9 | 保密标注 | 已明确标注"内部使用，不提交法院" |
| 4.2.10 | 回答禁区 | 明确哪些话不能说（不主动承认不利事实/不猜测对方主观意图/不评论法官态度） |

### 4.3 庭审纪要

| 序号 | 检查项 | 合格标准 |

| 8.3.4 | 当事人地位 | 当事人地位是否正确（按异议之诉类型调整原告/被告/第三人） |
| 8.3.5 | 诉讼请求 | 诉讼请求是否围绕"足以排除强制执行的民事权益"展开 |
| 8.3.6 | 前置程序说明 | 起诉状是否包含前置程序说明（已提出异议并收到裁定） |
| 8.3.7 | 互斥规则 | 是否排除了互斥情形（未同时启动第三人撤销之诉，《民事诉讼法解释》第301条） |
| 8.3.8 | 证据方向 | 证据是否围绕实体权利展开（权利取得时间、权利性质、占有/登记状态等） |
| 8.3.9 | 法条引用 | 法条引用是否准确（2023年版《民事诉讼法》及执行异议复议规定第28-31条） |
| 8.3.10 | 执行不停止 | 已提示异议之诉审理期间原则上不停止执行 |
| 8.3.11 | 分配方案异议之诉 | 如为分配方案异议之诉，已提醒一审终审（判决不可上诉） |

### 8.4 执行衍生与特殊救济检查

| 序号 | 检查项 | 合格标准 |
|------|--------|---------|
| 8.4.1 | 执行回转前提 | 必须执行完毕+执行依据被撤销，缺一不可 |
| 8.4.2 | 回转标的 | 回转标的明确（已执行的财产及其孳息） |
| 8.4.3 | 参与分配限制 | 仅适用于公民或其他组织，企业法人应申请破产 |
| 8.4.4 | 参与分配时间 | 在执行程序开始后、财产执行终结前提出 |
| 8.4.5 | 执行救助性质 | 是国家司法救助，不是执行程序的延续，不影响发现财产后恢复执行 |
| 8.4.6 | 救助金额 | 各地标准不同，常见为几万元，已提示用户 |
| 8.4.7 | 执行协调 | 是法院内部程序，一般由执行法院依职权或依当事人申请报请上级法院 |

## 九、类案检索质量检查红线

### 9.1 强制要求

| 序号 | 检查项 | 合格标准 | 不通过后果 |
|------|--------|---------|-----------|
| 9.1.1 | 实际检索执行 | AI生成类案检索报告前，**必须**执行实际检索（调用法律数据库（华宇元典法律数据库 yuandian_law）） | 不通过 |
| 9.1.2 | 专业数据库优先 | 优先检索人民法院案例库、裁判文书网、北大法宝、法信等专业数据库 | 不通过 |
| 9.1.3 | 检索执行记录 | 必须记录检索执行记录（平台、关键词、时间、结果数量） | 不通过 |
| 9.1.4 | 案例来源URL | 所有案例必须有可核实的来源URL | 不通过 |
| 9.1.5 | 不利案例检索 | 必须检索不利案例，只找有利案例会导致策略失误 | 不通过 |
| 9.1.6 | 无编造声明 | 报告头部或尾部必须有"严禁编造案例"的声明 | 不通过 |
| 9.1.7 | 检索工具证明 | 必须证明使用了法律数据库（华宇元典法律数据库 yuandian_law）或专业数据库进行实际检索 | 不通过 |

### 9.2 严禁行为（一票否决）

| 序号 | 严禁行为 | 处理方式 |
|------|---------|---------|
| 9.2.1 | 未检索而编造案例 | **不通过**，必须删除或标注"待核实" |
| 9.2.2 | 将"推断的典型案例"伪装为真实检索结果 | **不通过** |
| 9.2.3 | 填写虚假URL或不可核实的来源 | **不通过** |
| 9.2.4 | 未检索到案例时虚构填充 | **不通过**（涉嫌编造） |
| 9.2.5 | 仅检索有利案例而忽略不利案例 | **不通过** |
| 9.2.6 | 案例信息不完整（无案号/无法院/无裁判要旨） | **不通过** |
| 9.2.7 | 引用已废止或修订前的旧法条 | **不通过** |
| 9.2.8 | 凭记忆引用法条而不经数据库核实 | **不通过** |

### 9.3 未检索到案例时的正确处理

| 情形 | 正确处理 |
|------|---------|
| 经实际检索未找到相关类案 | 如实告知用户"经检索未找到相关类案" |
| 提供替代分析 | 提供法律原则分析和学理观点作为替代 |
| 提醒补充检索 | 提醒用户在专业数据库中补充深度检索 |
| 报告格式 | 报告仍应包含检索执行记录（证明已执行检索）和检索目标说明 |

### 9.4 法条引用未核实时的正确处理

| 情形 | 正确处理 |
|------|---------|
| 经实际检索未确认法条现行有效性 | 如实告知用户"经检索未确认该法条现行有效性，需律师人工核实" |
| 提供替代引用 | 提供基于训练数据的法条引用框架，明确标注"未经数据库核实，需人工确认" |
| 提醒补充核实 | 提醒用户在专业数据库中核实最新文本 |
| 标注方式 | 在法条汇编中标注"⚠️ 未经数据库核实，需人工确认" |

## 十、质量检查执行流程

### 10.1 自动检查

每个阶段产出完成后，agent自动执行：
1. 读取本阶段产出文件
2. 对照本文件中的检查清单逐项检查
3. 生成检查结果

### 10.2 检查结果处理

| 结果 | 处理方式 |
|------|---------|
| 全部通过 | 向用户汇报"本阶段质量检查通过" |
| 部分不通过 | 列出不通过项，自动修正后重新检查 |
| 严重不通过 | 向用户说明问题，请求补充材料或信息 |

### 10.3 检查报告格式

```markdown
# 质量检查报告

## 检查阶段：[程序]·[阶段]
## 检查时间：[YYYY-MM-DD]

## 检查结果
| 序号 | 检查项 | 结果 | 说明 |
|------|--------|------|------|

## 总体评价
- 通过项数：X/Y
- 不通过项数：Z
- 是否允许进入下一阶段：[是/否]

## 待修正项
1. [项] → [修正措施]

## 建议
```

### 10.4 不合格项处理规则

| 不合格项类型 | 处理方式 |
|-------------|---------|
| 材料缺失 | 列入待补充材料清单，标注优先级和截止日期 |
| 期限超期风险 | 立即提醒用户，评估补救措施（如申请顺延、申请恢复等） |
| 法条引用错误 | 重新检索确认，替换为准确条文 |
| 证据目录格式错误 | 按模板重新排版 |
| 内部材料混入证据 | 立即移出，重新编排证据册 |
| 遗漏质证 | 补充质证意见，重新检查覆盖范围 |
| 类案检索编造嫌疑 | 删除无法核实的案例，重新执行实际检索 |
| 再审事由不匹配 | 重新对照第211条/第212条逐项核对 |
| 执行期限错过 | 评估补救可能性，告知用户法律后果 |

## 十一、特殊案由质量检查汇总

### 11.1 各案由专属检查项速查

| 案由 | 关键检查项 |
|------|-----------|
| 债权人代位权 | 书面催告凭证/次债务人资格/债权范围限定/债务人未破产/非专属债权 |
| 债权人撤销权 | 除斥期间/撤销+返还双重请求/相对人恶意/生效文书确认评估/必要费用请求 |
| 人格否认 | 五维混同证据/诉讼地位正确/一人公司举证倒置/关联关系时点/穷尽常规救济 |
| 股东出资 | 章程核实/实际出资查明/加速到期触发条件/管辖确认/抽逃路径完整/案由区分 |
| 执行异议之诉 | 前置裁定/15日起诉期限/执行法院管辖/权利形成时间早于查封/实体权利证据 |

### 11.2 程序转换关键检查点

| 转换节点 | 关键检查 |
|---------|---------|
| 一审→二审 | 上诉期15日/10日未届满/上诉状已提交/上诉费已预缴 |
| 一审→执行 | 判决已生效/履行期届满/申请执行期2年内/执行依据明确 |
| 二审→再审 | 判决生效后6个月内/符合第211条事由/管辖正确 |
| 二审→执行 | 二审判决生效/履行期届满/申请执行期2年内 |
| 执行→异议之诉 | 收到裁定之日起15日内/执行法院管辖/前置程序完成 |
| 执行→复议 | 收到裁定之日起10日内/向上一级法院/仅针对行为异议裁定 |
| 再审→检察监督 | 驳回裁定/再审裁判生效之日起2年内/向同级检察院 |
| 再审→上诉 | 按一审程序再审的，判决书送达之日起15日内 |

# 01 战略把脉：从混乱到清晰的案件启动指南

> 核心理念：这个阶段用户最迷茫。不要急着写文书，先把材料理清楚、事实盘明白、法律搞透彻。磨刀不误砍柴工。

## Step 0：材料收集与整理（最重要的基础）

### 0.1 材料存放位置
用户应将案件材料放入当前文件夹的 `01-原始案件材料/` 子文件夹中。

### 0.2 AI协助盘点流程
读取 `01-原始案件材料/` 中的所有文件后：

1. **列出已识别材料清单** — 分类统计用户已提供的材料
2. **评估完整性** — 对照 02-参考模板/00-材料收集清单.md 逐项检查
3. **指出缺失材料** — 标注"必须补充"/"建议补充"/"可有可无"
4. **给出补救建议** — 对于缺失材料，告知如何补充（如申请调取、自行取证等）
5. **生成待补充材料清单** — 基于缺失材料，生成一份独立的《06-待补充材料清单.md》：缺什么、为什么需要、怎么获取、优先级、负责方、截止日期
6. **整理原始材料** — 在 `03-阶段产出/` 下创建 `原始案件材料（已整理）/` 文件夹，按类别创建子文件夹，将原始文件复制到对应子文件夹。原始文件保持不动
7. **创建补充材料文件夹** — 在 `03-阶段产出/` 下创建 `补充案件材料/` 空文件夹，供用户后续补充新材料
8. **生成材料索引与分类** — 将原始材料按类别编目，生成《01-材料索引与分类.md》。材料索引中简要引用待补充材料清单，详细内容在独立文件中

### 0.3 材料完整性评级
- **A级（材料齐全）**：基础材料全部具备，关键事实有证据支撑 → 进入下一步
- **B级（基本够用）**：缺少部分材料，但不影响核心事实认定 → 边整理边补充
- **C级（严重不足）**：关键证据缺失 → 优先解决材料补充问题

## Step 1：梳理案件事实 → 《案件事实时间轴》

### 1.1 操作指引
基于已收集的材料，按时间顺序提取关键事件。每个事件需要标注：
- 日期（精确到日，如无法确定则标注时间段）
- 事件内容（客观陈述，避免主观评价）
- 相关证据（材料名称及页码/位置）
- 性质（有利/不利/待核实/中性）
- 备注（需进一步核实的问题）

### 1.2 双版本标注
- **原告方（Builder）**：重点标注能证明对方违约/侵权的事实，形成完整证据链
- **被告方（Breaker）**：重点标注对方隐瞒的事实、逻辑漏洞、程序瑕疵

### 1.3 输出
生成《02-案件事实时间轴.md》存放于 `03-阶段产出/`，参考模板见 02-参考模板/01-案件事实时间轴模板.md

## Step 2：检索法律依据 → 《类案检索报告》+《法条汇编》

### 2.1 检索原则
- 先法条、后案例（避免重复检索）
- 优先检索近三年案例
- 检索顺位：指导性案例 → 公报案例 → 省高院参考案例 → 上级法院/本院裁判

### 2.2 类案检索（AI辅助 + 用户执行）

⚠️ **重要限制：AI检索能力有限，不能替代专业数据库的深度检索。**

AI可以做的：
1. **初步检索** — 通过互联网检索法院官网公开的典型案例、指导性案例、法官文章
2. **提炼检索目标** — 根据案情确定需要检索的争议法律问题
3. **设计检索策略** — 推荐关键词、检索平台、检索方法
4. **撰写报告格式** — 生成类案检索报告的框架和模板
5. **分析案例** — 对用户提供的裁判文书进行归纳、提炼、分析

AI不能保证的：
- 无法保证检索到所有相关案例
- 无法访问需要付费的专业数据库完整内容
- 无法实时访问裁判文书网的完整数据库

**用户必须补充完成**：
1. 在专业数据库（中国裁判文书网、北大法宝、MetaLaw等）中进行深度检索
2. 下载裁判文书PDF
3. 把裁判文书提供给AI进行分析完善

**所有AI检索到的案例必须标明来源URL，用户需自行复核真实性。严禁编造案例。**

生成《03-类案检索报告.md》存放于 `03-阶段产出/`，参考模板见 02-参考模板/02-类案检索报告模板.md

### 2.3 法条汇编
整理与本案相关的实体法、司法解释、程序法依据，生成《法条汇编.md》。

生成《04-法条汇编.md》存放于 `03-阶段产出/`，参考模板见 02-参考模板/03-法条汇编模板.md

## Step 3：制定诉讼方案 → 《诉讼/应诉策略方案书》

### 3.1 原告方（Builder）策略要点
- 挖掘"爽点"：还原履约过程，寻找对方违约/侵权的核心事实
- 寻找"请求权基础"：锁定案由，检索支持诉求的法条与类案
- 确定战术：是否申请财产保全？是否发律师函施压？诉求金额如何计算？

### 3.2 被告方（Breaker）策略要点
- 寻找"痛点"：核对原告诉状，寻找隐瞒事实、逻辑漏洞或诉讼时效问题
- 寻找"抗辩权基础"：检索免责条款、减责情形及程序瑕疵（如管辖错误）
- 确定防线：全面否认还是自认部分？是否提起反诉或管辖异议？

### 3.3 输出
生成《05-诉讼策略方案书.md》存放于 `03-阶段产出/`，参考模板见 02-参考模板/04-诉讼策略方案书模板.md

## 质量检查表（60分标准）

| 检查项 | 合格标准 |
|--------|---------|
| 材料收集 | 基础材料齐全，关键证据不缺失 |
| 材料整理 | 原始材料已按类别编目，索引清晰，方便后续阶段取用 |
| 待补充材料清单 | 独立文件，明确列出缺失材料、获取方法、优先级、负责方和截止日期 |
| 时间轴 | 覆盖所有关键日期，每个事实有证据支撑，区分有利/不利 |
| 类案检索 | 包含支持和不支持双方的案例，裁判要旨提炼准确，注明案号和法院 |
| 法条汇编 | 引用现行有效法律，条文内容准确，标注适用要点 |
| 策略方案 | 目标明确、路径清晰、有风险提示和Plan B，区分原告/被告视角 |

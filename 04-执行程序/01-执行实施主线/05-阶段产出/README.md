# 05-阶段产出

存放执行实施主线阶段 AI 生成的工作成果文件。

## 产出文件清单

| 序号 | 文件名称 | 触发分支 | 说明 |
|------|---------|---------|------|
| 01-强制执行申请书.md | 分支A：申请执行 | 申请执行的核心文书 |
| 02-财产查控措施申请书.md | 分支B：财产查控 | 针对具体财产的查封/冻结/扣押申请 |
| 03-执行和解协议.md | 分支C：执行和解 | 双方和解协议（不具有强制执行力） |
| 04-执行进程跟踪表.md | 分支F：执行跟踪 | 全程跟踪执行各节点状态 |
| 05-悬赏执行申请书.md | 分支E：悬赏执行 | 悬赏查找被执行人或财产 |
| 06-限制消费申请书.md | 分支D：信用惩戒 | 申请对被执行人限制消费 |
| 07-纳入失信名单申请书.md | 分支D：信用惩戒 | 申请将被执行人纳入失信被执行人名单 |
| 08-追加被执行人申请书.md | — | 申请追加/变更被执行人（如适用） |

## 组织方式

建议按执行进程的时间顺序编号存放，如：
```
05-阶段产出/
├── 01-强制执行申请书.md
├── 02-财产查控措施申请书.md
├── 03-财产线索清单.md
├── 04-执行进程跟踪表.md
└── ...
```

> ⚠️ 所有 AI 生成的文书均为**初稿，需律师审核后使用**。

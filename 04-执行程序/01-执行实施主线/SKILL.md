# 执行程序·10 执行实施主线 SKILL

## 触发条件

用户表达以下意图时触发：
- "申请强制执行"
- "帮我写强制执行申请书"
- "执行案件怎么推进"
- "被执行人没钱怎么办"
- "执行和解协议怎么写"
- "申请限制消费/纳入失信"

## 前置依赖

1. 已有生效法律文书（判决书/调解书/仲裁裁决/公证债权文书）
2. 法律文书确定的履行期限已届满，或对方明确表示不履行
3. 在申请执行期限内（2年）

## 工作流

### 0.1 读取输入

1. `01-执行依据/` 中的生效法律文书
2. `02-财产线索/` 中的财产线索材料（如有）
3. `04-执行法院来文/` 中的法院文书（如有）

### 0.2 申请执行期限检查

从法律文书中提取：
- 生效日期
- 履行期限（如有）
- 计算申请执行截止日期（生效/履行期届满后2年内）

如期限即将届满（≤3个月），立即提醒用户尽快申请执行。

### 0.3 执行依据类型确认

确认执行依据类型：
- 法院判决书/调解书
- 仲裁裁决书
- 公证债权文书
- 其他生效法律文书

不同类型可能有不同的执行规则和管辖要求。

## 工作流分支

### 分支A：申请执行

#### Step 1: 确定管辖法院

- 一般规则：一审法院或与一审法院同级的被执行财产所在地法院
- 仲裁裁决：被执行人住所地或财产所在地中级法院
- 公证债权文书：被执行人住所地或财产所在地法院

#### Step 2: 撰写强制执行申请书

生成 `05-阶段产出/01-强制执行申请书.md`

```
申请执行人：【姓名/名称】
被执行人：【姓名/名称】

申请事项：
1. 强制被执行人履行【生效法律文书案号】确定的义务，向申请执行人支付人民币【金额】元；
2. 强制被执行人支付迟延履行期间的债务利息（如适用）；
3. 强制被执行人承担本案执行费用。

事实与理由：
【生效法律文书的基本情况、履行情况、被执行人未履行的事实】

此致
【执行法院名称】

申请执行人：【签章】
日期：
```

#### Step 3: 整理申请材料清单

列出申请执行需提交的全部材料，标注已有/缺失。

### 分支B：财产查控措施申请

#### Step 1: 财产线索评估

读取 `02-财产线索/` 中的材料，评估：
- 财产类型（存款/房产/车辆/股权/债权等）
- 财产所在地
- 财产权属状况
- 是否已被其他法院查封

#### Step 2: 确定查控措施

根据财产类型确定控制措施：

| 财产类型 | 控制措施 | 申请文书 |
|---------|---------|---------|
| 银行存款 | 冻结、扣划 | 财产查控措施申请书 |
| 房产 | 查封 | 财产查控措施申请书 |
| 车辆 | 查封、扣押 | 财产查控措施申请书 |
| 股权 | 冻结 | 财产查控措施申请书 |
| 到期债权 | 冻结 | 财产查控措施申请书 |
| 知识产权 | 冻结 | 财产查控措施申请书 |

#### Step 3: 生成申请书

生成 `05-阶段产出/02-财产查控措施申请书.md`

### 分支C：执行和解协议

#### Step 1: 和解方案设计

如双方有和解意向，协助设计和解方案：
- 履行金额（可低于执行标的）
- 履行期限（一次性/分期）
- 担保措施（人保/物保）
- 违约责任（不履行则恢复原执行）
- 执行费用承担

#### Step 2: 风险提示

向用户提示执行和解的风险：
- 执行和解协议不具有强制执行力
- 对方不履行时需申请恢复执行
- 恢复执行受2年期限限制（从和解协议约定履行期的最后一日起算）

#### Step 3: 生成协议

生成 `05-阶段产出/03-执行和解协议.md`

### 分支D：信用惩戒措施申请

#### 限制消费申请

适用条件：被执行人未按执行通知书指定的期间履行生效法律文书确定的给付义务

生成 `05-阶段产出/06-限制消费申请书.md`

#### 纳入失信名单申请

适用条件（满足之一）：
1. 有履行能力而拒不履行生效法律文书确定义务
2. 以伪造证据、暴力、威胁等方法妨碍、抗拒执行
3. 以虚假诉讼、虚假仲裁或者以隐匿、转移财产等方法规避执行
4. 违反财产报告制度
5. 违反限制消费令
6. 无正当理由拒不履行执行和解协议

生成 `05-阶段产出/07-纳入失信名单申请书.md`

### 分支E：悬赏执行申请

适用条件：被执行人下落不明或财产难以查找

生成 `05-阶段产出/05-悬赏执行申请书.md`

### 分支F：执行进程跟踪

建立并维护 `05-阶段产出/04-执行进程跟踪表.md`，跟踪：

| 节点 | 状态 | 日期 | 备注 |
|------|------|------|------|
| 申请执行 | 已提交/已立案 | | |
| 发出执行通知 | 已完成/待完成 | | |
| 网络查控 | 已反馈 | | 查控结果摘要 |
| 财产控制 | 已查封/已冻结 | | 控制标的详情 |
| 财产处置 | 拍卖中/已成交 | | |
| 执行款发放 | 已发放 | | |
| 执行结果 | 完毕/终本/和解 | | |

## 质量检查清单

生成全部文件后，逐项核对：

- [ ] 执行依据是否已生效且可执行（判决书/调解书/仲裁裁决等）
- [ ] 申请执行期限是否在2年内（从法律文书规定履行期间的最后一日起算）
- [ ] 管辖法院是否正确（一般为一审法院或被执行财产所在地法院）
- [ ] 申请事项是否明确、具体、可执行（金额、行为、交付财物等）
- [ ] 财产线索描述是否准确（财产类型、所在地、权属状况、是否已被查封）
- [ ] 查控措施是否与财产类型匹配（冻结存款/查封房产/扣押车辆等）
- [ ] 信用惩戒申请是否符合法定条件（限消/失信的适用条件已核实）
- [ ] 执行和解协议是否包含风险提示（不具有强制执行力、恢复执行2年期限）
- [ ] 执行进程跟踪表是否包含关键节点（立案→查控→控制→处置→发放→结果）
- [ ] 全部文件编码为 UTF-8，中文显示正常
- [ ] 所有文件生成后标注"初稿，需律师审核后使用"

## 输出规范

- 强制执行申请书：请求事项明确、具体、可执行
- 财产查控申请书：财产线索描述准确，控制措施与财产类型匹配
- 执行和解协议：条款完整，风险提示清晰
- 信用惩戒申请书：符合法定条件，事实陈述准确
- 执行进程跟踪表：实时更新，关键节点不遗漏
- 所有文件生成后标注"初稿，需律师审核"

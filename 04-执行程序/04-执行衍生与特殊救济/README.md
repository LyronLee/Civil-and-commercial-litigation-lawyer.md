# 04-执行衍生与特殊救济

本模块处理四种执行衍生程序与特殊救济：执行回转、参与分配、执行协调、执行救助。

## 适用情形

| 子类型 | 适用情形 | 进入目录 |
|--------|---------|---------|
| 执行回转 | 执行完毕后，执行依据被撤销或变更 | `01-执行回转/` |
| 参与分配 | 多个债权人对同一被执行人，财产不足清偿 | `02-参与分配/` |
| 执行协调 | 多个法院执行冲突，需上级法院协调 | `03-执行协调/` |
| 执行救助 | 终本无财产，申请人生活困难 | `04-执行救助/` |

## 与执行异议之诉的区别

| 对比项 | 执行异议之诉 | 执行衍生与特殊救济 |
|--------|------------|------------------|
| 性质 | 诉讼程序 | 执行程序或司法救助 |
| 是否立案 | 需要起诉立案 | 一般无需立案（执行回转除外） |
| 是否有庭审 | 有 | 无 |
| 是否可上诉 | 可上诉（分配方案异议之诉除外） | 不适用 |

## 使用方式

1. 确定具体子类型（参考上表）
2. 进入对应子目录，阅读 `README.md` 和 `SKILL.md`
3. 按 `SKILL.md` 工作流逐步执行

## 通用规则

- 各子类型均配备**独立完整的四阶段工作流**
- 不跨程序引用其他程序框架
- 所有文件生成后标注"初稿，需律师审核后使用"

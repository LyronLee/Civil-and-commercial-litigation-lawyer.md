# 02 材料制备：把上诉策略转化为法院文书

> **核心理念**：阶段一想清楚，阶段二写清楚。本阶段不创造新的事实分析，而是把阶段一的结论转化为格式规范、可直接提交二审法院的法律文书。二审文书的核心是"原审哪里错了"，不是"事实本来怎样"。

## 前置依赖

1. 阶段一已完成，或用户直接跳过阶段一（需读取一审判决书和策略方案）
2. `01-上一阶段产出/` 中已放入阶段一产出（如阶段一已完成）
3. 如为被上诉人，`02-对方来文/` 中已放入对方上诉状

## 输入读取

### 上诉人模式输入
1. `01-上一阶段产出/01-判决书分析意见.md`
2. `01-上一阶段产出/02-上诉策略方案书.md`
3. 一审判决书（从一审卷宗中读取）
4. 一审起诉状（了解一审主张）
5. `02-上诉新证据/` 中的新材料（如有）

### 被上诉人模式输入
1. `02-对方来文/` 中的对方上诉状
2. 一审判决书
3. 一审答辩状（了解一审抗辩）
4. 一审证据目录
5. `01-上一阶段产出/` 中的策略材料（如有）

---

## Step 0：期限提取与跟踪

读取全部法院来文（`04-法院来文/`），提取并跟踪以下期限：

| 期限来源 | 期限事项 | 法定基准 | 优先级 |
|---------|---------|---------|--------|
| 应诉通知书 | 答辩期 | 15日 | 法院指定 > 法定 |
| 举证通知书 | 举证期限 | 法院指定（通常 ≥30日） | 法院指定 > 法定 |
| 传票 | 开庭日期 | 法院指定 | 法院指定 |
| 缴费通知书 | 上诉费预缴 | 上诉期内或法院指定期限 | 法院指定 > 法定 |
| 判决书 | 上诉期 | 判决15日 / 裁定10日 | 法院指定 > 法定 |

**规则**：
1. 如法院来文载明特定期限，以法院指定为准
2. 如法院未指定或指定不明，适用法定期限
3. 将提取的期限写入《关键期限跟踪表.md》，附于本阶段全部产出文件
4. 关键期限到达前3日提醒用户

---

## Step 1：角色确认 + 已有上诉状检测

### 1.1 角色确认

确认用户角色，决定生成上诉状还是二审答辩状：

| 角色 | 定义 | 生成文书 |
|------|------|---------|
| 上诉人 | 不服一审判决，提起上诉 | 《上诉状》（视检测结果决定是否生成） |
| 被上诉人 | 一审胜诉/部分胜诉，对方提起上诉 | 《二审答辩状》 |

**角色判定规则**（调用 `00-通用基础设施/角色判定规则.md` 第二章）：
1. 如策略方案书明确角色，按策略方案书执行
2. 如用户直接启动本阶段，询问确认
3. 如 `02-对方来文/` 中有对方上诉状但用户未明确角色，默认被上诉人模式

### 1.2 已有上诉状检测（★二审特有）

**上诉期限很短（判决书15日/裁定书10日），上诉状通常在一审收尾阶段即已完成。** 在决定生成上诉状前，必须先检测用户是否已有上诉状：

**检测路径**：
1. 扫描 `01-上一阶段产出/`（即 `../01-战略分析/04-阶段产出/`）中的文件
2. 扫描 `01-一审卷宗材料/` 及子目录
3. 扫描项目根目录及常见输出目录（如 `04-阶段产出/`、`06-阶段产出/`）
4. 扫描 `02-对方来文/`（检测对方是否已上诉）

**检测结果处理**：

| 检测情形 | 系统行为 |
|---------|---------|
| **检测到用户已有上诉状** | **询问用户**："检测到您已在一审阶段完成上诉状（文件路径：XXX），是否需要：①沿用现有上诉状；②修改现有上诉状；③重新起草？" 如选择①或②，跳过 Step 2.1（起草上诉状），直接进入 Step 2.2（新证据目录）及后续步骤 |
| **未检测到用户上诉状，但在上诉期内** | 询问用户："未检测到已有上诉状，您是否需要在二审阶段生成上诉状？" 用户确认需要时，才进入 Step 2.1 生成上诉状 |
| **未检测到用户上诉状，且上诉期已届满** | 提示用户上诉权已丧失，建议转入 `03-再审程序/`；如用户仍坚持，仅生成上诉状供参考（需明确告知逾期风险） |
| **检测到对方上诉状（被上诉人模式）** | 直接进入被上诉人工作流（Step 3），无需生成上诉状 |

> ⚠️ **核心原则**：本阶段**默认不自动生成上诉状**。仅在以下情形才生成：
> 1. 用户明确要求生成上诉状；
> 2. 未检测到已有上诉状，且用户确认需要生成；
> 3. 检测到已有上诉状，但用户选择"重新起草"。
>
> 避免在一审已完成上诉状的情况下重复劳动。

## 上诉人模式工作流

### Step 2.1：起草上诉状（条件触发）

**触发条件**：通过 Step 1.2 检测后，确认需要生成上诉状。

基于 `01-判决书分析意见.md` 和 `02-上诉策略方案书.md`，生成 `05-阶段产出/01-上诉状.md`。

#### 上诉状结构

```markdown
上诉人：【姓名/名称】
住所地/住址：
法定代表人/负责人：（如为法人）

被上诉人：【姓名/名称】
住所地/住址：

（原审第三人：如有，列明）

上诉请求：
1. 请求撤销【一审法院名称】（【案号】）民事判决第【X】项，改判【具体请求内容】；
2. 请求撤销一审判决，发回【一审法院名称】重审；
3. 请求判令被上诉人承担本案一、二审诉讼费用。

上诉理由：

一、原审判决认定事实错误（或事实不清）
【具体指出原审认定错误的某个事实，引用证据反驳】

二、原审判决适用法律错误
【具体指出原审引用的法条错误或不完整，引用正确的法律依据】

三、原审违反法定程序
【如存在程序问题，具体说明】

综上所述，原审判决【错误之处】，恳请二审法院依法改判。

此致
【二审法院名称】

上诉人：【签章】
日期：

附：
1. 本上诉状副本【X】份
2. 新证据目录及证据材料（如有）
```

#### 撰写要点

1. **上诉请求必须具体**：
   - 不能只写"请求撤销原判"，必须写明"撤销后怎么改"
   - 如请求发回重审，需说明理由（事实不清/程序违法）
   - 如为部分上诉，明确列明仅对判决第X项不服

2. **上诉理由聚焦重点**：
   - 选择策略方案书中确定的1-3个核心争点
   - 每个争点独立成段，逻辑清晰
   - 引用一审判决原文 + 反驳理由 + 正确观点

3. **不重复一审论述**：
   - 上诉状不是"加强版起诉状"
   - 重点在于"原审哪里错了"，不是"事实本来怎样"
   - 避免把一审已经讲过的内容再讲一遍

4. **引用规范**：
   - 引用一审判决原文时标注页码或段落
   - 引用法条注明全称和条文号
   - 引用证据标注证据编号和页码

5. **部分上诉的特殊处理**：
   - 首部明确"上诉人仅对一审判决第X项不服，提起上诉"
   - 上诉请求仅针对不服的部分
   - 对其他判项明确表示服判，避免二审扩大审理范围

---

### Step 2.2：制作新证据目录（如有）

如有二审新证据，生成 `05-阶段产出/02-新证据目录.md`。

**格式**：同证据目录（序号/名称/证明目的/页码），但每份证据需额外标注：

| 额外字段 | 说明 |
|---------|------|
| 是否属于新证据 | 依据民诉法司法解释，说明属于哪种新证据类型 |
| 一审未提交原因 | 客观原因说明（如新发现、一审申请调取未获准等） |
| 对上诉请求的支持作用 | 该证据针对一审判决的哪个错误 |

---

### Step 2.3：生成辅助文书（按需）

#### 上诉费预缴说明

生成 `05-阶段产出/03-上诉费计算表.md`，包含：
- 不服一审判决部分的标的额
- 按《诉讼费用交纳办法》分段计算的上诉案件受理费
- 缴费期限提醒
- 缴费方式（法院指定账户/诉讼费专窗）

#### 调查取证申请书

如需要二审法院调取一审未调取的证据：
- 需调取的具体证据名称和内容
- 证据持有方名称
- 一审中申请调取未获准许的裁定（如有）
- 该证据对二审改判的重要性说明

#### 待补充材料清单

阶段二视角的缺失信息，列入 `05-阶段产出/04-待补充材料清单.md`：
- 上诉状中引用的证据是否已整理归档
- 新证据是否已准备原件核对
- 上诉费是否已计算并准备缴纳

## 被上诉人模式工作流

### Step 3.1：起草二审答辩状

基于对方上诉状，生成 `05-阶段产出/01-二审答辩状.md`。

#### 答辩状结构

```markdown
答辩人：【姓名/名称】
住所地/住址：

上诉人：【姓名/名称】

答辩人因与上诉人【案由】纠纷一案，针对上诉人的上诉请求和理由，答辩如下：

一、针对上诉请求一的答辩
【逐条反驳】

二、针对上诉理由的答辩
【分点反驳对方的上诉理由】

三、一审判决认定事实清楚、适用法律正确
【正面阐述一审判决的正确性】

综上所述，请求二审法院驳回上诉，维持原判。

此致
【二审法院名称】

答辩人：【签章】
日期：

附：
1. 本答辩状副本【X】份
2. 证据目录及证据材料（如有反驳新证据）
```

#### 撰写要点

1. **逐条对应反驳**：对方上诉状有几条理由，答辩状就逐条反驳
2. **维护一审判决**：强调一审判决认定事实清楚、适用法律正确
3. **可适当主动出击**：如对方上诉理由暴露了新漏洞，可主动指出
4. **不引入新争议**：二审答辩状以防守为主，避免节外生枝
5. **反诉的预留**：如客户情况说明中勾选了"反诉"，在答辩状中预留反诉空间，并提示用户另行起草反诉状

## 质量检查表（阶段二完成标准）

生成全部文件后，逐项核对：

- [ ] 角色已明确确认（上诉人/被上诉人）
- [ ] 上诉状/答辩状格式规范，首部信息完整
- [ ] 上诉请求具体明确，不模糊（写明"撤销后怎么改"）
- [ ] 上诉理由聚焦1-3个核心争点，不面面俱到
- [ ] 引用一审判决原文时标注了页码或段落
- [ ] 法条引用完整准确，注明全称和条文号
- [ ] 如为部分上诉，已明确列明仅对哪些判项不服
- [ ] 新证据目录（如有）已标注新证据类型和一审未提交原因
- [ ] 上诉费已计算并写入缴费说明
- [ ] 关键期限已提取并写入跟踪表（答辩期、举证期、开庭日、缴费期）
- [ ] 来文管理文件夹已创建，用户知晓用途
- [ ] 所有文书已标注"初稿，需律师审核后使用"

## 通用原则

1. **原始材料保护**：AI 不移动、不修改、不删除原始文件
2. **双版本设计**：关键文书区分"内部工作版"与"提交法院版"
3. **AI 边界声明**：所有生成文书标注"初稿，需律师审核后使用"
4. **严禁编造**：引用法条、案例必须准确
5. **期限优先**：法院指定 > 法定期限，关键期限到达前主动提醒

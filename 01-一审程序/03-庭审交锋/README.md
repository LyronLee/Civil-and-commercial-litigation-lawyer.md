# 03-庭审交锋

> **定位**：收到对方诉讼材料后完成质证，开庭前完成庭审准备，庭审后整理纪要。
> **工作时机**：对方提交答辩状、证据目录及证据材料后；开庭前；庭审结束后。

## 关键设计原则

1. **卷宗索引，不重复搬运**：本阶段通过路径引用阶段二的完整卷宗，保持单份文件、避免冗余。
2. **质证即攻防**：质证意见不是单向的"挑对方毛病"，而是双向的"攻防转换"——对方证据有利时主动拿来强化我方观点。
3. **开庭提纲是内部作战地图**：包含敏感策略预判（如对方可能的抗辩路径、我方底线），仅限内部使用。
4. **庭审纪要及时闭环**：开庭后 24 小时内完成，记录法官口头要求（可能不在书面裁定中体现），防止遗漏。

## 特殊案由庭审要点

| 案由 | 质证/庭审特殊关注点 |
|------|-------------------|
| 债权人代位权纠纷 | ① 债务人对次债务人的债权是否已到期；② 债务人是否"怠于"行使权利（有无诉讼/仲裁）；③ 代位金额是否超出原告债权额 |
| 债权人撤销权纠纷 | ① 债权成立时间是否早于处分行为；② 处分行为是否"明显不合理低价"（需评估报告）；③ 相对人是否明知；④ 除斥期间是否届满 |
| 损害公司债权人利益责任纠纷 | ① 人格混同证据（财务账册、人员交叉、业务混同）；② 因果关系（混同行为→不能清偿）；③ 一人公司财产独立举证责任倒置 |
| 股东出资纠纷 | ① 加速到期的触发条件（终本裁定/延期出资/破产原因）；② 出资瑕疵的时间节点；③ 抽逃出资的路径链条 |
| 执行异议之诉 | ① 权利形成时间是否早于查封；② 权利性质是否足以排除执行；③ 执行标的描述是否准确 |

> 💡 上述要点已内置于 `SKILL.md` 中，AI在生成质证意见和开庭提纲时会自动关注对应案由的特殊争点。

## 调解相关准备

### 开庭提纲中的调解预案

开庭提纲模板已内置"调解预案"板块（见 `01-参考模板/01-开庭提纲模板.md`）。AI生成时会自动包含：
- 我方调解底线（金额/条件/期限）
- 最优调解方案（比判决更有利的条件）
- 调解话术准备
- 当事人授权确认

### 庭审中的调解

如庭审中法官组织调解：
1. 确认调解是否在己方授权范围内（如超出，需联系当事人确认）
2. 使用已准备的调解预案进行谈判
3. 调解达成一致 → 法院制作调解笔录 → 签收调解书
4. 调解书生效后 → 直接跳至阶段四的结案复盘

### 庭审纪要中的调解记录

如庭审中进行了调解，无论是否达成一致，庭审纪要中需记录：
- 调解过程摘要
- 双方提出的条件
- 是否达成一致
- 如未达成一致，分歧点在哪里
- 下次调解安排（如有）

> 💡 **调解时机**：庭前调解和庭中调解是常见时机。通常法官在法庭调查结束后、辩论开始前会询问调解意愿。做好准备，不要措手不及。

# 案件事实时间轴模板

> 按时间顺序梳理案件经过，标注每个事实的证据支撑和对己方的影响。
>
> 原则：**客观陈述，避免主观评价**。忠实于材料和证据，不夸大、不隐瞒。

---

## 头部信息

```markdown
# 案件事实时间轴

## 案件基本信息
- **案由**：（如：民间借贷纠纷、买卖合同纠纷）
- **案号**：（如已进入诉讼程序）
- **我方诉讼地位**：原告 / 被告
- **对方当事人**：（姓名/名称）
- **受理法院**：（如已知）
- **制作日期**：
```

**图例说明**：
- 🟢 有利 — 对己方有利的事实
- 🔴 不利 — 对己方不利的事实（需准备解释或应对方案）

## 填写指引

### 日期
- 尽量精确到**具体日期**（如：2023年5月12日）
- 无法确定具体日期的，标注**时间段**（如：2023年5月中下旬）
- 顺序有争议的，标注"约"并说明理由

### 事件内容
- **客观陈述**，不加主观评价
- 示例：
  - ✅ "甲向乙转账人民币10万元，备注'借款'"
  - ❌ "甲好心借钱给乙"（主观评价）
- 涉及多人的，明确**行为主体**（谁做了什么）

### 相关证据
- 注明证据**名称和位置**
- 示例："《借款合同》第2页"、"微信聊天记录_20230512_截图3"
- 无直接证据的事实，标注"待补充证据"

### 性质（关键！）

#### 原告方标注指南

| 标记 | 含义 | 示例 |
|------|------|------|
| 🟢 有利 | 证明对方违约/侵权/欠债的事实 | 合同约定交货日期为X月X日，实际未交货 |
| 🔴 不利 | 可能削弱己方主张的事实 | 己方也曾延迟付款；己方口头同意变更合同 |
| 🟡 待核实 | 事实不清，需要进一步确认 | 对方声称已发货，但己方未收到 |
| ⚪ 中性 | 背景信息，与争议焦点关系不大 | 双方相识多年，合作多次 |

#### 被告方标注指南

| 标记 | 含义 | 示例 |
|------|------|------|
| 🟢 有利 | 证明原告主张不成立或存在漏洞的事实 | 原告提供的合同系伪造；已过诉讼时效 |
| 🔴 不利 | 承认对己方不利但需解释或减轻责任的事实 | 确实延迟交货，但有不可抗力理由 |
| 🟡 待核实 | 原告声称的事实需要核实 | 原告声称损失50万，实际损失待评估 |
| ⚪ 中性 | 背景信息 | 双方长期合作关系 |

## 五、时间轴可视化建议

如果时间跨度长、事件多，建议在表格之外，再制作一个**可视化时间轴**：

```
2023.01    2023.03    2023.05    2023.07    2023.09
  |          |          |          |          |
  ▼          ▼          ▼          ▼          ▼
签合同    首次付款    对方违约    发送催告    提起诉讼
```

可以使用PPT、ProcessOn、Visio等工具制作，附在时间轴文档后面。

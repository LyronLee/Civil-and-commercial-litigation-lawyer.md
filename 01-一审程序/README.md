# 01 一审程序：民事诉讼的起点

> 🎯 **程序定位**：民事诉讼的第一审程序，覆盖从接案、材料收集、策略制定、文书制备、庭审交锋到一审判决/调解/撤诉的全部流程。
>
> **一审是民事诉讼的起点**：各程序（一审、二审、再审、执行）均配备**独立完整的四阶段工作流**，一审程序的框架设计经验为其他程序提供了参考范式。

---

## 什么时候进入一审程序

| 情形 | 说明 |
|------|------|
| 刚接到一个新案件 | ✅ 从阶段一开始 |
| 收到法院传票（被告） | ✅ 直接进入阶段一，分析对方材料 |
| 一审判决已生效 | ❌ 一审已结束，如需救济→02-二审/03-再审/04-执行 |

## 与其他程序的衔接

### 一审结束后的路径选择

| 一审结果 | 我方角色 | 后续程序 |
|---------|---------|---------|
| 胜诉，对方履行 | 原告 | 时点四：结案复盘（无需进入其他程序） |
| 胜诉，对方不履行 | 原告 | → **04-执行程序** |
| 败诉 | 原告/被告 | → **02-二审程序**（判决15日内） |
| 部分胜诉 | 任何一方 | 综合评估 → 02-二审 或 04-执行 |
| 调解结案 | 任何一方 | 调解书生效 → 对方不履行 → **04-执行程序** |
| 撤诉 | 原告 | 如需要可重新起诉 |

### 和解/调解成功的快捷路径

如在一审任何阶段达成和解/调解：
- **阶段二达成和解**（起诉前/立案后）：可撤诉或申请法院出具调解书 → 直接进入阶段四的结案复盘
- **阶段三达成调解**（庭审中）：法院出具调解书 → 阶段四仅做结案复盘
- **阶段四达成和解**（判决前）：原告撤诉或双方申请调解 → 结案复盘

## 特殊案由适配

如案件属于以下特殊案由，框架会自动调用适配模板：

| 案由 | 适配内容 |
|------|---------|
| 债权人代位权纠纷 | 被告=次债务人、管辖=次债务人住所地、诉讼请求限定 |
| 债权人撤销权纠纷 | 共同被告=债务人和相对人、双重请求（撤销+返还）、除斥期间 |
| 损害公司债权人利益责任纠纷 | 三种当事人组合、连带责任、人格混同证据 |
| 股东出资纠纷 | 加速到期触发条件、补充赔偿责任、出资瑕疵证据 |
| 执行异议之诉 | 四类当事人组合、专属管辖、前置程序 |

> 💡 使用方式与正常案由完全相同，AI 会自动识别并适配。也可在 `01-原始案件材料/` 中填写 `特殊情况说明.md` 辅助识别。

## 快速开始

### 第一步：进入阶段一

```bash
cd 01-一审程序/01-战略把脉/
```

### 第二步：放入材料

将当事人提供的所有材料放入 `01-原始案件材料/`。

### 第三步：唤起 AI

> **"我在做01一审程序的01战略把脉阶段，请帮我分析这个案子。材料在01-原始案件材料里。"**

AI 会引导你完成四个阶段的工作。

## 关键期限提醒

| 事项 | 期限 |
|------|------|
| 答辩期 | 15日（收到应诉通知书起） |
| 举证期限 | 法院指定（通常≥30日） |
| 上诉期 | 判决15日 / 裁定10日 |
| 申请执行期 | 判决生效后2年 |

> ⏰ **规则**：法院指定的期限优先于法定期限。收到法院文书后，务必以文书载明的期限为准。

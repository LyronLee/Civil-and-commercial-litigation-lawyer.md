四、关键设计决策（后续阶段必须遵循）
================================================================================

【1. 文件夹命名规范】
- 阶段文件夹：01-战略把脉、02-诉讼材料及来文管理、03-庭审交锋、04-收官与执行
- 阶段内子文件夹编号：01-输入区、02-工具区、03-产出区（阶段一特殊：01-原始案件材料）
- 产出文件编号：01-07（md文件）
- 非编号文件夹：放在03-阶段产出/内，用中文描述性名称

【2. 阶段产出放在最后原则】
- 03-阶段产出（或06-阶段产出）是阶段内最后一个编号文件夹
- 包含所有AI产出（文书+整理材料+补充材料入口）
- 可直接复制到下一阶段使用

【3. 原始材料保护原则】
- 阶段一的 01-原始案件材料/ 里的文件保持原样不动
- AI不移动、不修改、不删除原始文件，只复制整理
- 阶段二不直接读取阶段一目录，通过用户复制到 01-上一阶段产出/ 衔接

【4. AI能力边界声明（特别重要）】
- AI可以通过互联网初步检索：法院官网公开的典型案例、指导性案例、法官文章、部分公开裁判文书
- AI不能保证检索到所有相关案例，无法访问付费数据库完整内容
- AI严禁编造案例，类案检索采用"AI初步检索 + 用户深度检索补充"模式
- AI检索到的案例必须标明来源URL，用户需自行复核真实性
- AI可以：初步检索、提炼检索目标、设计策略、撰写格式、分析用户提供的案例
- 用户必须：在专业数据库进行深度检索补充、下载裁判文书、提供给AI分析
- **阶段二新增**：AI生成的起诉状/答辩状/证据目录等法律文书均为初稿，必须经律师审核后才能提交法院

【5. 双版本设计原则】
- 所有模板区分原告（Builder）和被告（Breaker）视角
- 阶段二SKILL内置角色判定逻辑和模式切换
- 关键节点提供双版本检查清单

【6. 产出文件命名】
- SKILL.md 文件名固定（Kimi识别要求），name字段用中文
- 产出md文件统一编号：01-、02-...
- 参考模板文件编号：00-05
- 01-起诉状和01-答辩状互斥，根据角色生成其一

【7. 阶段三特殊设计决策】
- 不复制上一阶段产出，直接通过路径引用阶段二卷宗
- 质证意见从阶段二的条件触发skill移至阶段三作为固定产出
- 质证意见策略性增强：攻防一体、借力打力、预先设防、强化锚点
- 庭审纪要中"下次开庭安排"改为"其他安排"，范围扩大至补充材料、法官要求、下次开庭等

【8. 阶段四特殊设计决策】
- 从"两个时点"扩展为"四个时点"：时点一（代理词）、时点二（裁判应对）、时点三（退费）、时点四（结案与复盘）
- 工作流 B（裁判应对与结案）拆分为工作流 B（裁判应对）、C（退费）、D（结案与复盘）
- 新增复盘功能：AI辅助生成初稿+律师/当事人手写补充，经验沉淀为可复用方法论
- 新增"法院来文"文件夹（02-），接收判决书、裁定书、生效证明等
- 新增"退费申请"文件夹（03-），管理诉讼费退费材料
- 裁判应对二选一：上诉状和强制执行申请书互斥，根据判决结果自动生成其一
- 卷宗引用范围扩大至前三个阶段的全部产出

【SKILL.md 三步工作流】
Step 0：读取卷宗，确认案件状态（读取阶段二全部材料，确认角色、对方材料、法院文书、开庭时间）
Step 1：撰写质证意见（逐份四性分析 + 策略性评价：对我方不利则削弱，对我方有利则转化利用）
Step 2：撰写开庭提纲（程序核对、法官发问预判、我方发问设计、争议焦点、辩论要点、突发应对）
Step 3：庭审后整理庭审纪要（基本信息、程序事项、调查摘要、辩论摘要、其他安排）

【质证意见策略性设计】
- 攻防一体：质证不是单向挑错，对方证据有利时主动转化为我方论据
- 借力打力：对方用来证明A的证据，若同时证明B（我方主张），在质证中"反转"为我方论据
- 预先设防：预判对方庭审中如何引用该证据攻击我方，提前埋入反驳逻辑
- 强化锚点：每份证据质证后，如有可能用一句话回扣我方核心主张

【福景花园案例阶段三产出】

- 02-开庭提纲.md：程序核对清单、6个法官发问预判（附回答口径+禁区+延伸应对）、5个我方发问设计（附目的+预期+追问+风险提示）、5个争议焦点梳理、两轮辩论要点+最后陈述、突发应对预案表、代理词备忘

【SKILL.md 四个独立工作流】
工作流 A（时点一）：庭后代理词
  Step 0：读取庭审卷宗（阶段三产出）
  Step 1：撰写代理词（庭后补充提交，系统梳理事实和法律，强化法官心证）
工作流 B（时点二）：裁判应对
  Step 0：读取全部卷宗，确认判决结果、上诉期限、履行期限
  Step 1：裁判应对（二选一：不服判决→上诉状；胜诉后对方不履行→强制执行申请书）
工作流 C（时点三）：诉讼费退费
  Step 0：读取判决书/缴费凭证，确认诉讼费承担方式
  Step 1：生成退费申请书
工作流 D（时点四）：结案与复盘
  Step 0：读取阶段一至四全部材料
  Step 1：撰写结案报告（案件结果、文书归档清单、后续事项）
  Step 2：撰写复盘总结（AI自动分析+律师/当事人手写补充，经验沉淀为方法论）

【复盘功能设计】
- AI视角：基于完整卷宗自动生成阶段工作评估（完成度评分）、成功经验、不足之处、改进建议、类案沉淀
- 人工补充区：律师/当事人手写补充主观感受、客户关系反思、下次优先改进事项
- 类案处理经验沉淀：启动清单、关键证据清单、常见抗辩及应对、时间节奏参考

【福景花园案例阶段四产出（预审前版）】

九、质量检查标准
================================================================================

阶段一检查项：
- [ ] 材料收集：基础材料齐全，关键证据不缺失
- [ ] 材料整理：原始材料已按类别编目，索引清晰
- [ ] 待补充材料清单：独立文件，明确缺失材料和获取方法
- [ ] 时间轴：覆盖所有关键日期，每个事实有证据支撑
- [ ] 类案检索：包含真实案例（或已明确检索策略）
- [ ] 法条汇编：引用现行有效法律
- [ ] 策略方案：目标明确、路径清晰、有风险提示和Plan B

阶段二检查项：
- [ ] 角色确认：已明确原告/被告方向，客户情况说明已填写
- [ ] 输入完整性：01-上一阶段产出/ 已包含策略方案书、材料索引、时间轴、法条汇编
- [ ] 起诉状/答辩状：格式规范、请求/抗辩明确、事实有证据支撑、引用法条准确
- [ ] 证据目录：4列格式、按争议焦点分组、编号清晰（证据X-Y）、页码标注
- [ ] 证据说明：每份证据有形式说明、内容摘要、关联性分析、证明力评估
- [ ] 诉讼费用：计算准确、费用项目完整
- [ ] 辅助文书：保全/调查取证申请书按需生成，内容完整
- [ ] 待补充清单：阶段二视角的缺失信息已列出，有获取方法、优先级、负责方
- [ ] 来文管理：04-对方来文/ 和 05-法院来文/ 已创建，用户知晓用途
- [ ] AI边界声明：所有生成的文书已标注"初稿，需律师审核后使用"

阶段三检查项：
- [ ] 卷宗引用：AI已正确读取阶段二的起诉状、证据目录、证据说明、对方来文、法院来文
- [ ] 质证意见：覆盖对方全部证据，无遗漏；包含四性分析；包含策略性评价（有利证据的转化利用）
- [ ] 开庭提纲：涵盖程序核对、发问预判、我方发问、争议焦点、辩论要点、突发应对
- [ ] 庭审纪要：模板完整，尚未开庭时输出空模板即可
- [ ] 内部文件标注：开庭提纲和庭审纪要已标注"内部使用，不提交法院"

阶段四检查项：
- [ ] 卷宗引用：AI已正确读取阶段一至三的全部产出和本阶段法院来文
- [ ] 代理词：基于庭审事实，无新的事实主张，系统梳理法律适用
- [ ] 裁判应对：上诉状和强制执行申请书二选一，不会同时生成
- [ ] 退费申请：包含缴费凭证、判决书、收款账户信息
- [ ] 结案报告：完整记录案件经过、结果、归档清单、后续事项
- [ ] 复盘总结：包含AI自动分析（阶段评估、成功经验、不足、改进建议、类案沉淀）和人工补充区
- [ ] 上诉期限提醒：已提示用户上诉期限（判决15日/裁定10日）
- [ ] 执行期限提醒：已提示用户申请执行期限（二年）

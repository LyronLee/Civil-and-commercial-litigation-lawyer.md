# 使用指南

> 本项目是一个**人与 AI 协作的诉讼工作框架**。要使用它，你只需要三样东西：
> 1. 一个能读取长文本的 AI 工具
> 2. 本项目的 `SKILL.md` 文件（工作流指引）
> 3. 你的案件材料
>
> **最简单的用法**：下载本项目 → 打开对应阶段的 `SKILL.md` → 复制粘贴到 AI 对话中 → 按步骤执行。不需要任何配置。

## 第一步：准备项目

### 下载本项目

```bash
# 方式一：git 克隆
git clone https://github.com/yourname/民事诉讼律师.git 民事诉讼律师.md

# 方式二：直接下载 ZIP
# 在 GitHub 页面点击 "Code" → "Download ZIP"，解压到本地
```

### 了解项目结构

下载后你会看到四个阶段文件夹：

```
民事诉讼律师.md/
├── 01-战略把脉/           ← 接案后，整理材料、分析案件
├── 02-诉讼材料及来文管理/  ← 起草起诉状/答辩状、制作证据目录
├── 03-庭审交锋/           ← 收到对方材料后，写质证意见、准备开庭
├── 04-收官与执行/         ← 开完庭后写代理词，收到判决后应对
├── 使用指南.md            ← 本文件
└── README.md              ← 项目介绍
```

每个阶段文件夹里都有：
- `README.md` — 这个阶段的说明（给人看的）
- `SKILL.md` — AI 工作流指引（给 AI 看的，你把它发给 AI 即可）
- `参考模板/` — 空模板，供你和 AI 参考

**你只需要做一件事**：按顺序进入每个阶段，把该阶段的 `SKILL.md` 内容发给 AI，然后按 AI 的指引配合完成工作。

### 📄 关于 Markdown 格式

本项目全部文件使用 **Markdown（`.md`）** 格式。如果你不熟悉，不用担心——它本质上就是带格式的纯文本，比 Word 更简单：

- **打开方式**：任何文本编辑器（记事本、VS Code、Typora）都能直接打开
- **AI 最懂它**：Markdown 的结构（标题、列表、表格）对 AI 来说最容易识别，这让 AI 能更准确地执行工作流
- **随时转 Word**：需要提交法院的正式文书，可以一键转换：
  - **Typora / Obsidian**：直接导出 `.docx`
  - **Pandoc**（命令行）：`pandoc 起诉状.md -o 起诉状.docx`
  - **手动复制**：全选 → 复制 → 粘贴到 Word 即可

| 推荐的 Markdown 阅读器 | 平台 | 特点 |
|------------------------|------|------|
| **Typora** | Win/Mac/Linux | 所见即所得，支持导出 Word/PDF，最推荐 |
| **Obsidian** | 全平台 | 本地优先，适合管理多个案件 |
| **VS Code** | Win/Mac/Linux | 免费开源，开发者首选 |
| **系统记事本** | 任何平台 | 无需安装，直接阅读 |

## 强烈推荐：Kimi Code CLI

[Kimi Code CLI](https://github.com/MoonshotAI/kimi-cli) 是月之暗面推出的命令行 AI 助手。**作者强烈建议使用这个工具**，原因如下：

1. **直接读取本地文件**：AI 能自动打开并分析你电脑上的案件材料，不需要手动复制粘贴
2. **长文本支持**：Kimi 的上下文窗口达 200K，足以处理完整的 SKILL.md + 大量案件材料
3. **文件上传便捷**：支持 `Ctrl+U` 快速上传文件，支持拖拽文件夹
4. **中文理解能力强**：对法律文书的格式、术语理解准确

**使用 Kimi Code CLI 不需要配置 Skill 目录**，只需安装后在对话中上传/粘贴 SKILL.md 即可。下面的安装步骤是为了获得命令行交互能力。

### 第一步：安装 Kimi Code CLI

**Linux / Mac：**

```bash
# 需要 Python 3.10+
pip install kimi-cli

# 或者使用 uv
uv tool install kimi-cli
```

**Windows：**

```powershell
# 需要 Python 3.10+
pip install kimi-cli
```

> 💡 **Windows 执行策略解锁**：如果在 PowerShell 中遇到 "无法加载文件，因为在此系统上禁止运行脚本" 错误，请执行：
> ```powershell
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> # 然后重新运行 pip install
> ```
> 这是使用 pip 安装命令行工具的常规操作，仅影响当前用户。

验证安装：

```bash
kimi --version
```

### 第二步：配置 API Key

```bash
# 启动 Kimi Code CLI，按提示完成 API Key 配置
kimi
```

或者在配置文件中直接设置：

- **Linux/Mac**：`~/.config/kimi/config.toml`
- **Windows**：`%APPDATA%\kimi\config.toml`

```toml
[provider]
name = "moonshot"
api_key = "your-moonshot-api-key"
model = "kimi-k2"
```

> 获取 API Key：[Moonshot AI 开放平台](https://platform.moonshot.cn/)

### 第三步：开始使用

```bash
# 进入本项目目录
cd 民事诉讼律师.md

# 启动 Kimi Code CLI
kimi
```

进入对话后，直接上传 SKILL.md 文件（按 `Ctrl+U`）：

```
> 请阅读我上传的 SKILL.md 文件，按其中的工作流帮我完成阶段一的任务。
> 我的案件材料在 01-原始案件材料/ 文件夹中。
```

或者复制粘贴：

```
> [粘贴 SKILL.md 的完整内容]
>
> 请按以上工作流帮我完成阶段二的任务。
> 阶段一的产出在 01-上一阶段产出/ 中。
```

Kimi Code CLI 的优势在于可以直接读取你本地的文件路径，AI 能自动打开并分析指定文件夹中的材料。

## 其他方式

如果你不想安装命令行工具，以下方式同样可以使用本项目。

### 方式 A：Kimi 网页版 / App（无需安装）

1. 打开 [Kimi 智能助手](https://kimi.moonshot.cn/) 或 Kimi App
2. 新建一个对话
3. **上传 SKILL.md 文件**：点击附件按钮，选择对应阶段的 `SKILL.md` 文件
4. **发送提示语**：
   ```
   请按照上传的 SKILL.md 中的工作流，帮我完成 [具体任务]。
   我的案件材料如下：[描述案件情况或上传材料]
   ```

### 方式 B：ChatGPT / Claude / 其他 AI 工具（通用）

1. 打开你喜欢的 AI 工具（ChatGPT、Claude、Gemini 等）
2. 新建对话
3. **复制 SKILL.md 内容**：打开对应阶段的 `SKILL.md` 文件，复制全部内容
4. **粘贴到对话中**，然后发送你的案件信息

**提示语模板**：

```
【以下是我使用的专业工作流指引，请严格按照其中的步骤和要求执行】

[粘贴 SKILL.md 的完整内容]

---

【我的案件信息】

阶段：[一/二/三/四]
角色：[原告/被告]
案件类型：[合同纠纷/侵权纠纷/劳动争议等]

[描述案件情况或上传材料]

请按照上述工作流，帮我完成本阶段的任务。
```

> ⚠️ **注意**：ChatGPT 免费版有对话长度限制，如果 SKILL.md 内容过长，可能需要分段粘贴，或者升级到 GPT-4/Plus 版本。Claude 的长文本处理能力较强，通常可以一次性处理完整的 SKILL.md。

## 完整使用流程示例

以下是一个完整的案件处理流程，使用 **Kimi Code CLI**（直接上传 SKILL.md）：

### Step 1：接案后 — 阶段一（战略把脉）

```bash
# 启动 Kimi Code CLI
cd 民事诉讼律师.md
kimi
```

```
> 请阅读我上传的 SKILL.md（01-战略把脉/SKILL.md）。
> 我刚接了一个买卖合同纠纷案子，当事人是原告，卖方收了定金没交货。
> 材料在 01-原始案件材料/ 里面，请帮我整理分析。
# 按 Ctrl+U 上传 01-战略把脉/SKILL.md
```

AI 按 SKILL.md 中的工作流执行，生成：
- `01-战略把脉/03-阶段产出/01-材料索引与分类.md`
- `01-战略把脉/03-阶段产出/02-案件事实时间轴.md`
- `01-战略把脉/03-阶段产出/03-类案检索报告.md`
- `01-战略把脉/03-阶段产出/04-法条汇编.md`
- `01-战略把脉/03-阶段产出/05-诉讼策略方案书.md`

**专业人士审核**这些产出，补充类案检索、调整策略方向。

### Step 2：确定诉讼方向后 — 阶段二（诉讼材料）

```bash
# 复制阶段一产出到阶段二（Linux/Mac）
cp -r 01-战略把脉/03-阶段产出/* 02-诉讼材料及来文管理/01-上一阶段产出/
# Windows: xcopy "01-战略把脉\03-阶段产出\*" "02-诉讼材料及来文管理\01-上一阶段产出\" /E /I
```

让当事人填写 `02-诉讼材料及来文管理/02-客户情况说明/诉讼意图情况说明.md`。

```
> 请阅读我上传的 SKILL.md（02-诉讼材料及来文管理/SKILL.md）。
> 阶段一已完成，材料已复制到 01-上一阶段产出/。
> 客户情况说明也已填写。我是原告方，请帮我起草起诉状、证据目录和证据说明。
# 按 Ctrl+U 上传 02-诉讼材料及来文管理/SKILL.md
```

AI 生成起诉状、证据目录、证据说明等。

**专业人士审核修改后**，打印/盖章/提交法院。

### Step 3：收到对方材料后 — 阶段三（庭审交锋）

将对方提交的答辩状、证据材料放入 `02-诉讼材料及来文管理/04-对方来文/`。
将法院送达的传票、举证通知书等放入 `02-诉讼材料及来文管理/05-法院来文/`。

```
> 请阅读我上传的 SKILL.md（03-庭审交锋/SKILL.md）。
> 对方提交了答辩状和 5 份证据，法院也发了传票。
> 请帮我写质证意见和开庭提纲，准备庭审。
# 按 Ctrl+U 上传 03-庭审交锋/SKILL.md
```

AI 生成质证意见和开庭提纲。

**质证意见审核后提交法院**；开庭提纲打印出来庭审时参考。

### Step 4A：开完庭后 — 阶段四（时点一：代理词）

```
> 请阅读我上传的 SKILL.md（04-收官与执行/SKILL.md）。
> 开完庭了，请帮我写一份代理词，提交给法院。
# 按 Ctrl+U 上传 04-收官与执行/SKILL.md
```

AI 生成代理词。

**审核后尽快提交法院**（判决作出前）。

### Step 4B：收到判决书后 — 阶段四（时点二：裁判应对）

将判决书放入 `04-收官与执行/02-法院来文/`。

```
> 请阅读我上传的 SKILL.md（04-收官与执行/SKILL.md）。
> 收到判决书了，判决部分胜诉，对方需赔偿但违约金请求被驳回。
> 请帮我看看是否需要上诉，还是申请强制执行。
# 按 Ctrl+U 上传 04-收官与执行/SKILL.md
```

AI 生成上诉状或强制执行申请书（二选一）。

**专业人士审核后执行**：决定是否上诉/申请执行。

### Step 4C：判决生效后 — 阶段四（时点三：退费）

```
> 判决生效了，对方没有上诉。请帮我写退费申请书。
```

AI 生成退费申请书。

### Step 4D：案件终局后 — 阶段四（时点四：结案与复盘）

案件彻底结束后（履行完毕或执行完毕）：

```
> 案子结了，请帮我写结案报告和复盘总结。
```

AI 生成结案报告和复盘总结。

**归档复盘**：将结案报告和复盘总结存档，沉淀经验。

## 常见问题

### Q1：我没有 Kimi Code CLI，能用这个项目吗？

**可以。** 本项目不绑定任何特定 AI 工具。你可以使用 Kimi 网页版、ChatGPT、Claude 等任何支持长文本处理的 AI 工具。只需将对应阶段的 `SKILL.md` 文件内容复制粘贴到对话中即可。Kimi Code CLI 是作者推荐的工具，但不是必须的。

### Q2：SKILL.md 文件很长，AI 读不完怎么办？

大多数现代 AI（GPT-4、Claude 3.5/4、Kimi）的上下文窗口都在 128K 到 200K 之间，足以处理本项目任何一个阶段的 SKILL.md（约 200-400 行）。如果你使用的是较早版本的模型，可以：
1. 分段粘贴 SKILL.md 内容
2. 升级到支持更长上下文的模型版本
3. 使用 Kimi Code CLI，它的上下文管理能力更强

### Q3：我是当事人，没有律师，能直接用这个项目自己打官司吗？

**强烈不建议。** 本项目生成的所有文书均为**初稿**，需要具备法律专业背景的人士审核后才能提交法院。诉讼涉及复杂的法律判断和策略选择，AI 无法替代律师的专业能力。当事人可以在**律师的指导下**使用本项目整理材料、了解流程，但不应自行提交未经审核的文书。

### Q4：每个阶段都要新建对话吗？

**建议每个阶段新建一个对话。** 原因：
- SKILL.md 每个阶段内容不同，新建对话可以避免混淆
- 阶段二的输出很长（起诉状+证据目录+证据说明），保留在单独对话中便于后续查找
- 如果你使用的是有对话长度限制的 AI，分阶段可以避免超出上下文限制

当然，如果你使用的 AI 上下文足够长（如 Claude 3.5/4、Kimi），也可以在一个对话中连续完成多个阶段，只需在切换阶段时重新发送对应的 SKILL.md 即可。

### Q5：我的案件材料涉及隐私，上传到 AI 安全吗？

**请务必注意数据安全。**
- 使用任何在线 AI 工具时，数据都会发送到该服务商的服务器
- **建议**：在上传前对敏感信息（身份证号、银行账户、具体金额等）进行脱敏处理，或用代号代替
- 对于高度敏感的案件，建议仅让 AI 指导你手动撰写，不上传原始材料

### Q6：AI 生成的文书可以直接提交法院吗？

**不可以。** 所有 AI 生成的文书都标注了"初稿，需审核后使用"。在提交法院前，必须由执业律师或具备法律专业背景的人士进行以下审核：
- 事实陈述是否准确
- 法律引用是否正确
- 诉讼请求是否合理
- 证据链条是否完整
- 是否符合受理法院的格式要求

### Q7：Windows 下安装 Kimi Code CLI 遇到问题？

**问题 1：pip 不是内部或外部命令**
- 解决：确保 Python 已安装，并将 Python 的 Scripts 目录添加到系统 PATH 中

**问题 2：PowerShell 执行策略限制**
```powershell
# 错误信息：无法加载文件，因为在此系统上禁止运行脚本
# 解决：在 PowerShell 中执行
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
# 输入 Y 确认
```

**问题 3：中文字符显示乱码**
- 这是 Windows 终端的编码问题，不影响文件内容
- 解决：在 PowerShell 中执行 `chcp 65001` 切换到 UTF-8 编码

### Q8：这个项目支持哪些案由？

当前版本为通用框架，**理论上支持所有民事案由**。实际测试中主要围绕合同纠纷、建筑物区分所有权纠纷、侵权责任纠纷等常见类型。对于特殊案由（如海事海商、知识产权、涉外民事等），可能需要根据实际情况调整模板内容。

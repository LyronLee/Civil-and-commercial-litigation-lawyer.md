# 使用指南

> **V1.0.0**
>
> 本项目是一个**单Skill民事诉讼AI辅助工作框架**。要使用它，你只需要两样东西：
> 1. 一个支持Kimi Skill的AI工具（Kimi Code CLI / Kimi 桌面端）
> 2. 你的案件材料
>
> **最简单的用法**：安装本Skill → 告诉AI"我有一个案子，材料在XX文件夹" → AI自动完成全部工作。

## 第一步：安装Skill

### ✅ 直接上传文件夹安装（最傻瓜化，强烈推荐）

> 无需打开文件，无需复制粘贴，直接把文件夹丢给Kimi即可。

**步骤**：

1. **下载本项目**
   - GitHub页面点击 "Code" → "Download ZIP"
   - 解压ZIP文件到任意位置

2. **打开 Kimi 对话框**
   - Kimi Code CLI / Kimi 桌面端 均可

3. **上传整个项目文件夹**
   - 拖拽解压后的 `民事诉讼律师.md - V1.0.0/` 文件夹到对话框
   - 或点击附件按钮，选择整个文件夹

4. **发送安装指令**
   ```
   请帮我安装这个Skill。Skill文件在 civil-litigation-lawyer 中。
   ```

5. **Kimi 自动完成安装**
   - Kimi 会自动读取文件夹中的 `civil-litigation-lawyer/SKILL.md`
   - 调用 SkillManage 工具完成安装
   - 安装完成后会提示"Skill 安装成功"

> 💡 **提示**：如果上传的是ZIP文件，Kimi会自动解压并找到Skill文件。

## 第二步：使用Skill

### 核心用法（一句话启动）

安装完成后，在对话中直接说：

```
我有一个案子，材料在 C:\Users\XXX\案件材料\ 文件夹里。基本案件情况是xxxxxx。我们想达到的起诉效果是XXXX。
```

AI会自动：
1. **识别案件状态** — 读取文件夹中的文件，判断是起诉阶段、已立案、已开庭、已判决、需上诉、需执行等
2. **进入Ask模式**（如信息不足）— 自动询问缺失的关键信息（当事人身份、诉讼意图、案件类型等）
3. **制定计划** — 生成 `plan.md`，明确阶段划分、任务清单、时间节点
4. **执行工作流** — 按阶段自动生成全部法律文书和内部工作文件

### 你也可以这样启动

| 场景 | 你说 |
|------|------|
| 准备起诉 | "我要起诉XX公司，材料在XX文件夹" |
| 收到传票 | "我收到法院传票了，材料在XX文件夹" |
| 收到判决书 | "一审判决下来了，帮我看看要不要上诉" |
| 判决生效 | "对方不履行判决，帮我申请执行" |
| 只有案情描述 | "我有一个买卖合同纠纷，对方收了定金没交货" |
| 信息不全 | "我想咨询一下这个案子该怎么打" |

### 信息收集（Ask模式）

如果案件材料不足或关键信息缺失，AI会自动进入Ask模式，按以下优先级询问：

**L1（必须回答）**：
- 当事人身份信息（姓名/名称、住所、联系方式）
- 案件类型/案由（合同纠纷/侵权纠纷/物权纠纷等）
- 诉讼地位（原告/被告/第三人）
- 争议标的（金额/行为/权利）
- 案件材料存放路径

**L2（强烈建议回答）**：
- 管辖法院（或合同履行地/侵权行为地）
- 是否已立案/收到传票/收到判决
- 是否有对方来文（起诉状/答辩状/证据）
- 是否有法院来文（传票/举证通知书/判决书）
- 是否接受调解
- 是否申请财产保全

**L3（可选补充）**：
- 特殊背景信息
- 关联案件信息
- 当事人特殊要求

> 💡 **提示**：即使信息不全，AI也会基于已有信息开始工作，并在过程中逐步补充。你不需要一次性提供所有信息。

## 完整使用流程示例

### 示例一：新接案件（准备起诉）

**用户**：
```
我有一个买卖合同纠纷案子，当事人是原告，卖方收了定金没交货。
材料在 D:\案件\2026-买卖合同纠纷-福景花园\ 文件夹里。
```

**AI自动执行**：

1. **读取材料** — 扫描文件夹中的所有文件（PDF/图片/Word/Excel）
2. **识别状态** — 无法院文书，有合同/协议/沟通记录 → 判断为"新案/准备起诉"
3. **进入阶段一** — 战略把脉
4. **生成产出**：
   - `01-材料索引与分类.md`
   - `02-案件事实时间轴.md`
   - `03-类案检索报告.md`（调用法律数据库检索）
   - `04-法条汇编.md`（调用法律数据库核实法条）
   - `05-诉讼策略方案书.md`
5. **质量检查** — 自动执行质量检查清单
6. **汇报结果** — 向用户汇报阶段一完成，提示进入阶段二

**用户确认后**：
```
请继续完成阶段二，起草起诉状和证据目录。
```

**AI继续执行阶段二**，生成起诉状、证据目录、证据说明等。

---

### 示例二：收到判决书后

**用户**：
```
收到一审判决书了，判决部分胜诉，对方需赔偿但违约金请求被驳回。
材料在 D:\案件\2026-买卖合同纠纷-福景花园\ 文件夹里。
```

**AI自动执行**：

1. **读取材料** — 发现判决书 → 判断为"一审已裁判"
2. **分析判决** — 提取判决主文、事实认定、本院认为、裁判依据
3. **评估上诉价值** — 分析被驳回的违约金请求是否有上诉空间
4. **生成产出**：
   - `判决书分析清单.md`
   - `上诉策略方案书.md`
   - `改判概率评估.md`
5. **汇报结果** — 建议是否上诉，并提醒上诉期限（15日）

**用户决定上诉后**：
```
决定上诉，请帮我起草上诉状。
```

**AI进入二审程序**，生成上诉状、二审证据目录等。

### 示例三：信息不全时

**用户**：
```
我想咨询一下，对方欠我钱不还，我该怎么办？
```

**AI自动进入Ask模式**：

```
您好，为了帮您制定最合适的诉讼策略，我需要了解以下信息：

【必须回答】
1. 对方是个人还是公司？名称是什么？
2. 欠款金额大约多少？
3. 是否有借条/合同/转账记录等证据？

【建议补充】
5. 是否有约定还款期限？是否已到期？
6. 最后一次催款是什么时候？
7. 对方是否有财产线索？

请尽量提供以上信息，如有材料可上传或告知存放路径。
```

## 常见问题

### Q1：我需要手动选择阶段吗？

**不需要。** AI会自动识别案件状态并进入正确阶段。你只需提供材料文件夹路径或描述案情即可。

### Q2：我需要复制项目结构吗？

**不需要。** V1.0.0是单Skill架构，所有模板、规则、工作流均内置于Skill中。AI会根据需要动态生成文件，无需手动复制任何文件夹。

### Q3：AI生成的文书可以直接提交法院吗？

**不可以。** 所有AI生成的文书均标注"初稿，需律师审核后使用"。提交法院前必须由执业律师审核、修改并签字/盖章。

### Q4：我的案件材料涉及隐私，安全吗？

**请注意数据安全。**
- 使用在线AI工具时，数据会发送到服务商服务器
- **建议**：上传前对敏感信息（身份证号、银行账户、具体金额等）进行脱敏处理，或用代号代替
- 对于高度敏感案件，建议仅让AI指导你手动撰写，不上传原始材料

### Q5：没有律师，我能自己用这个打官司吗？

**强烈不建议。** AI生成的是初稿，需要具备法律专业背景的人士审核后才能提交法院。诉讼涉及复杂的法律判断和策略选择，AI无法替代律师的专业能力。当事人可以在律师指导下使用本项目整理材料、了解流程。

### Q6：AI会编造案例或法条吗？

**不会。** V1.0.0强制要求AI在引用法条或检索案例前，调用专业法律数据库（华宇元典法律数据库）进行核实。如未检索到案例，AI会如实告知"未找到相关类案"，并提供法律原则分析作为替代。

### Q7：支持哪些案由？

当前版本支持**所有普通民事案由**（合同、侵权、物权、婚姻家庭、劳动争议等），并对以下**5种特殊案由**提供专属适配：

- 债权人代位权纠纷
- 债权人撤销权纠纷
- 损害公司债权人利益责任纠纷（人格否认）
- 股东出资纠纷（加速到期/抽逃出资）
- 执行异议之诉（4个子类型）

### Q8：怎么知道该进入哪个程序？

**不需要你知道。** 只需描述你的情况或提供材料，AI会自动判断：

| 你的情况 | AI自动进入 |
|---------|-----------|
| 刚接案，准备起诉或应诉 | 一审程序 · 阶段一 |
| 收到一审判决，不服 | 二审程序（提醒15日上诉期） |
| 判决已生效，认为有错误 | 再审程序（提醒6个月申请期） |
| 判决生效，对方不履行 | 执行程序（提醒2年执行期） |
| 对执行行为/标的有异议 | 执行审查程序 / 执行异议之诉 |

### Q9：每个阶段都要新建对话吗？

**建议在一个对话中连续完成。** 单Skill架构下，AI会记住案件上下文，连续执行多个阶段。如需切换案件，建议新建对话。

### Q10：Windows下使用遇到问题？

**问题1：Kimi Code CLI安装失败**
- 确保Python 3.10+已安装
- 执行 `pip install kimi-cli`

**问题2：Skill安装后未识别**
- 确认Skill文件夹已复制到正确的skills目录
- 重启Kimi Code CLI

**问题3：中文字符显示乱码**
- 在PowerShell中执行 `chcp 65001` 切换到UTF-8编码

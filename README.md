## 这是什么

`民事诉讼律师.md` 是一个**人与 AI 协作的诉讼工作框架**。它将民事诉讼的全流程拆解为四个阶段，每个阶段配备：

- **README.md** — 给使用者看的操作说明
- **SKILL.md** — 给 AI 看的工作流指引（Kimi / ChatGPT / Claude 均可识别）
- **参考模板** — 22 份可直接复用的 Markdown 模板
- **阶段产出区** — AI 生成文书的标准输出位置

使用者按阶段放入案件材料、唤起 AI、审核 AI 产出，即可完成从立案到结案的全部文书准备工作。涉及提交法院的正式文书，建议由具备法律执业资格的专业人士审核把关。

## 📄 为什么使用 Markdown 格式

本项目所有模板和产出文件均采用 **Markdown（`.md`）** 格式，这是一种轻量级标记语言，具有以下优势：

- **纯文本，永不过时**：不像 Word 或 PDF 依赖特定软件，Markdown 是纯文本，任何设备、任何系统都能打开
- **结构清晰，AI 友好**：标题、列表、表格等结构对 AI 来说非常容易识别和解析，这让 AI 能更准确地理解你的工作流和产出内容
- **版本控制友好**：配合 Git 使用，可以清晰追踪每一份文书的修改历史
- **专注内容，不纠结排版**：用简单的符号就能表达层级和重点，不会被字体、颜色分散注意力
- **一键转换**：可轻松导出为 Word、PDF、HTML 等任何格式

### 推荐的 Markdown 阅读器

| 工具 | 平台 | 特点 |
|------|------|------|
| **Typora** | Win/Mac/Linux | 所见即所得，支持导出 Word/PDF，最推荐 |
| **Obsidian** | Win/Mac/Linux/iOS/Android | 双向链接、本地优先，适合管理多个案件 |
| **VS Code** + Markdown 插件 | Win/Mac/Linux | 免费开源，配合 Git 使用最佳 |
| **MarkText** | Win/Mac/Linux | 开源免费，类似 Typora |
| **系统自带记事本** | 任何平台 | 无需安装，直接打开 `.md` 文件即可阅读 |

> 💡 **需要 Word 格式？** 你可以在以下任一环节将 Markdown 转换为 Word：
> 1. **Typora / Obsidian**：直接导出 `.docx`，格式保留完整
> 2. **Pandoc**（命令行工具）：`pandoc 起诉状.md -o 起诉状.docx`
> 3. **在线转换工具**：如 [md2pdf.netlify.app](https://md2pdf.netlify.app/) 等
> 4. **手动复制**：直接在 Markdown 阅读器中全选复制，粘贴到 Word 中即可
>
> 提交法院的正式文书，建议转换为 Word 后按法院要求的字体（通常宋体小四/四号）和格式调整。

## 快速开始

> 📖 **首次使用？请先看《使用指南.md》** — 包含 Kimi Code CLI 安装配置、三种使用方式（命令行/网页版/通用AI工具）、各阶段提示语示例、常见问题等完整教程。

### 环境准备

1. **下载本项目**到本地工作目录
2. **准备 AI 工具**：推荐使用 Kimi（本项目 SKILL.md 按 Kimi 的 skill 格式编写），ChatGPT / Claude 亦可使用
3. **准备一个空文件夹**作为案件工作区，将本项目复制进去

### 阶段一：战略把脉（接案后）

```bash
# 1. 进入阶段一
cd 01-战略把脉/

# 2. 将当事人提供的原始材料放入 01-原始案件材料/
#    支持 PDF、图片、Word、Excel 等格式

# 3. 唤起 AI，上传 SKILL.md，说：
#    "请帮我整理这个案件的材料，完成阶段一工作流"

# 4. AI 自动生成 03-阶段产出/ 中的全部文件
# 5. 律师审核 AI 产出，补充类案检索、调整策略方案
```

### 阶段二：诉讼材料及来文管理（确定诉讼方向后）

```bash
# 1. 将阶段一产出复制到阶段二
#    cp -r 01-战略把脉/03-阶段产出/* 02-诉讼材料及来文管理/01-上一阶段产出/

# 2. 让当事人填写 02-客户情况说明/诉讼意图情况说明.md

# 3. 唤起 AI，上传 SKILL.md，说：
#    "请基于阶段一产出和客户情况说明，完成阶段二工作流"

# 4. AI 自动生成起诉状/答辩状、证据目录、证据说明等
# 5. 专业人士审核修改后，打印/盖章/提交法院
```

### 阶段三：庭审交锋（收到对方材料后）

```bash
# 1. 将对方材料放入 02-诉讼材料及来文管理/04-对方来文/
#    将法院文书放入 02-诉讼材料及来文管理/05-法院来文/

# 2. 进入阶段三
#    cd 03-庭审交锋/

# 3. 唤起 AI，上传 SKILL.md，说：
#    "对方提交了答辩状和证据，请帮我完成质证意见和开庭提纲"

# 4. AI 自动读取阶段二卷宗，生成质证意见和开庭提纲
# 5. 质证意见审核后提交法院；开庭提纲为内部文件，不提交
```

### 阶段四：收官与执行（开完庭后 / 收到判决书后）

```bash
# 时点一：开完庭后
# 唤起 AI，说："开完庭了，请帮我写代理词"
# → 生成 04-阶段产出/01-代理词.md

# 时点二：收到判决书后
# 唤起 AI，说："收到判决了，请帮我看看怎么应对"
# → 根据判决结果生成上诉状 或 强制执行申请书
# → 同时生成退费申请书、结案报告、复盘总结
```

## 各阶段详解

### 阶段一：战略把脉 — "想清楚"

> 磨刀不误砍柴工。在动笔前，必须完成"事实-法律-策略"的三位一体。

| 步骤 | 核心任务 | 产出物 |
|------|---------|--------|
| 1.1 | 收集整理材料，评估完整性 | 《材料索引与分类》 |
| 1.2 | 梳理案件事实时间轴 | 《案件事实时间轴》 |
| 1.3 | 检索法律依据与类案 | 《类案检索报告》、《法条汇编》 |
| 1.4 | 制定诉讼/应诉策略 | **《诉讼策略方案书》** |

**关键设计**：AI 禁止编造案例。类案检索采用"AI 初步检索 + 用户在专业数据库深度补充"模式。

### 阶段二：诉讼材料及来文管理 — "写清楚"

> 将混乱的事实转化为严谨的法律语言。

| 步骤 | 核心任务 | 产出物 |
|------|---------|--------|
| 2.1 | 确认角色（原告/被告），读取客户情况说明 | — |
| 2.2 | 起草起诉状 / 答辩状 | 《起诉状》/《答辩状》 |
| 2.3 | 制作证据目录（4 列简化版） | 《证据目录》 |
| 2.4 | 撰写证据说明（双版本） | 《证据说明（自用版）》、《证据说明（提交法院版）》 |
| 2.5 | 生成辅助文书（按需） | 《诉讼费用计算表》、《财产保全申请书》、《调查取证申请书》 |

**关键设计**：
- 起诉状事实叙述客观陈述，**不穿插证据编号**
- 证据目录排除内部材料（案件背景说明等不纳入证据册）
- 证据册页码底部居中，双面打印友好
- 提交法院版证据说明采用段落式排版 + `>` 强调语句，便于书记员复制

### 阶段三：庭审交锋 — "打赢仗"

> 质证不是单向挑错，而是攻防转换。对方证据有利时，主动拿来强化我方观点。

| 步骤 | 核心任务 | 产出物 |
|------|---------|--------|
| 3.1 | 对对方证据逐一质证（真实性、合法性、关联性、证明力 + 策略性评价） | 《质证意见》 |
| 3.2 | 开庭前准备：法官发问预判、我方发问设计、争议焦点、辩论要点 | 《开庭提纲》（内部使用，不提交法院） |
| 3.3 | 庭审后整理：程序事项、调查摘要、辩论摘要、法官要求 | 《庭审纪要》（内部使用，不提交法院） |

**关键设计**：
- 质证意见包含"策略性评价"：对我方有利证据主动"借力打力"
- 开庭提纲包含"回答禁区"：明确哪些话不能说
- 庭审纪要采用稀疏排版，打印后方便手写补充

### 阶段四：收官与执行 — "沉住气"

> 庭审结束不等于案件结束。分四个时点推进：开完庭后先补充代理词；收到判决后处理裁判应对；判决生效后办理退费；案件终局后完成结案归档和复盘。

| 时机 | 触发条件 | 产出物 |
|------|---------|--------|
| 时点一 | 开完庭后 | 《代理词》 |
| 时点二 | 收到判决书后 | 《上诉状》/《强制执行申请书》（二选一） |
| 时点三 | 判决生效后 | 《退费申请书》 |
| 时点四 | 案件终局后 | 《结案报告》+《复盘总结》 |

**关键设计**：
- 复盘总结是阶段四的核心增值：AI 基于完整卷宗自动生成阶段评估、成功经验、不足之处、改进建议、类案方法论；律师/当事人手写补充主观经验
- 裁判应对二选一：败诉 → 上诉状；胜诉但对方不履行 → 强制执行申请书
- 退费申请单独管理，避免遗漏

## 设计原则

1. **原始材料保护**：AI 不移动、不修改、不删除原始文件，只复制整理。当事人的原始材料始终保存在 `01-原始案件材料/` 中。
2. **阶段产出放最后**：每个阶段的产出文件夹编号最大，便于直接识别和复制。
3. **双版本设计**：关键文书区分"内部工作版"（含策略预判、证明力评估）和"提交法院版"（正式、客观、便于复制）。
4. **AI 能力边界**：AI 生成的是**初稿**，须经律师审核后方可提交法院；AI 严禁编造案例；AI 无法访问付费法律数据库的完整内容。
5. **卷宗索引不搬运**：阶段三、四通过相对路径直接引用前一阶段卷宗，保持单份文件、避免冗余。
6. **原告 / 被告双模式**：阶段二内置角色判定逻辑，自动切换"建设型"（原告）和"拆弹型"（被告）工作流。

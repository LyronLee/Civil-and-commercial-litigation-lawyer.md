## 这是什么

`民事诉讼律师.md` 是一个**单Skill民事诉讼AI辅助工作框架**。V1.0.0将全流程内化于一个Skill中，用户只需说"我有一个案子"，agent自动完成其余一切。

### 核心理念

- **无需复制项目结构** — 所有模板、规则、工作流内置于Skill
- **无需手动选择阶段** — AI自动识别案件状态并路由到正确程序
- **无需上传多个SKILL.md** — 单入口，自动加载所需内容
- **信息不全也能启动** — 自动进入Ask模式，逐项收集缺失信息
- **法律数据库强制检索** — 引用法条和案例前必须调用专业数据库核实

### Skill文件组成

```
civil-litigation-lawyer/
├── SKILL.md              ← 主入口（触发条件、核心工作流、AI能力边界）
├── ask-protocol.md       ← 自动询问协议（L1/L2/L3信息模型）
├── content.md            ← 完整工作流（通用基础设施+一审/二审/再审/执行）
├── templates.md          ← 模板结构规范（60+份文档模板）
├── rules.md              ← 通用规则（案由适配/程序转换/角色判定/期限计算）
└── quality-check.md      ← 质量检查清单（各阶段检查项+红线）
```

> 📖 **首次使用？请先看《使用指南.md》** — 包含Skill安装方法、使用方式、各阶段提示语示例、常见问题等完整教程。

### 安装Skill

#### ✅ 直接上传文件夹安装（最傻瓜化，强烈推荐）

> 无需打开文件，无需复制粘贴，直接把文件夹丢给Kimi即可。

1. **下载本项目**
   - GitHub页面点击 "Code" → "Download ZIP"
   - 解压ZIP文件到任意位置

2. **打开 Kimi 对话框**
   - Kimi Code CLI / Kimi 桌面端 均可
   
3. **上传整个项目文件夹**
   - 拖拽解压后的 `民事诉讼律师.md - V1.0.0/` 文件夹到对话框
   - 或点击附件按钮，选择整个文件夹

4. **发送安装指令**
   ```
   请帮我安装这个Skill。Skill文件在 civil-litigation-lawyer 中。
   ```

5. **Kimi 自动完成安装**
   - Kimi 会自动读取文件夹中的 `civil-litigation-lawyer/SKILL.md`
   - 调用 SkillManage 工具完成安装
   - 安装完成后会提示"Skill 安装成功"

> 💡 **提示**：如果上传的是ZIP文件，Kimi会自动解压并找到Skill文件。

### 使用Skill（一句话启动）

```
我有一个案子，材料在 C:\Users\XXX\案件材料\ 文件夹里。基本案件情况是xxxxxx。我们想达到的起诉效果是XXXX。
```

AI自动：
1. 读取文件夹中的所有文件
2. 识别案件状态
3. 如信息不足，进入Ask模式询问

## 各程序速查

### 一审程序 — 四阶段骨架

> 一审程序是框架的核心，所有特殊案由都通过"插件"方式复用同一骨架。

**阶段一：战略把脉 — "想清楚"**

| 步骤 | 核心任务 | 产出物 |
|------|---------|--------|
| 0.5 | 案由识别与特殊情况检测 | 案由标注、特殊材料清单 |
| 1.1 | 收集整理材料，评估完整性 | 《材料索引与分类》 |
| 1.2 | 梳理案件事实时间轴 | 《案件事实时间轴》 |
| 1.3 | 检索法律依据与类案（强制数据库检索） | 《类案检索报告》、《法条汇编》 |
| 1.4 | 制定诉讼/应诉策略 | **《诉讼策略方案书》** |

**阶段二：诉讼材料及来文管理 — "写清楚"**

| 步骤 | 核心任务 | 产出物 |
|------|---------|--------|
| 2.1 | 确认角色，读取客户情况说明，检测特殊案由 | — |
| 2.2 | 起草起诉状 / 答辩状（特殊案由自动适配） | 《起诉状》/《答辩状》 |
| 2.3 | 制作证据目录 | 《证据目录》 |
| 2.4 | 撰写证据说明（双版本） | 《证据说明（自用版）》、《证据说明（提交法院版）》 |
| 2.5 | 生成辅助文书（按需） | 《诉讼费用计算表》、《财产保全申请书》等 |

**阶段三：庭审交锋 — "打赢仗"**

| 步骤 | 核心任务 | 产出物 |
|------|---------|--------|
| 3.1 | 对对方证据逐一质证（四性 + 策略性评价） | 《质证意见》 |
| 3.2 | 开庭前准备 | 《开庭提纲》 |
| 3.3 | 庭审后整理 | 《庭审纪要》 |

**阶段四：收官与执行 — "沉住气"**

| 时机 | 触发条件 | 产出物 | 后续路径 |
|------|---------|--------|---------|
| 时点一 | 开完庭后 | 《代理词》 | — |
| 时点二 | 调解/和解成功 | 调解书/和解协议 | 直接进入时点五（结案复盘） |
| 时点三 | 收到判决书后 | 《上诉状》/《强制执行申请书》 | 上诉→二审程序 / 执行→执行程序 |
| 时点四 | 判决生效后 | 《退费申请书》 | — |
| 时点五 | 案件终局后 | 《结案报告》+《复盘总结》 | — |

### 二审程序 — 独立四阶段工作流

二审程序配备独立完整的四阶段工作流，核心特点：
- **阶段一**：分析一审判决的错误点，评估改判可能性
- **阶段二**：起草上诉状/二审答辩状
- **阶段三**：二审庭审准备（审理范围限于上诉请求）
- **阶段四**：二审判决后的处理（再审/执行/结案）

### 再审程序 — 特殊救济，三阶段

- **阶段一（再审申请）**：评估是否符合民诉法第211条的再审事由，撰写再审申请书
- **阶段二（再审审理）**：法院裁定再审后，按原审程序重新审理，配备独立完整的四阶段工作流
- **阶段三（收官）**：分析再审裁判效力，选择后续路径（上诉/检察监督/执行回转/结案）

> ⚠️ 再审不是必然发生的程序。大多数再审申请被驳回。只有符合法定再审事由且在判决生效后6个月内申请的，才有可能进入再审审理。

### 执行程序 — 四大模块

| 模块 | 定位 | 核心内容 |
|------|------|---------|
| 01-执行实施主线 | 执行的核心 | 申请执行→财产查控→处置→完毕/终本 |
| 02-执行审查程序 | 执行中的救济 | 行为异议/案外人异议/复议/监督 |
| 03-执行异议之诉 | 实体争议的诉讼 | 四种异议之诉，独立诉讼类型，配备独立完整的四阶段工作流 |
| 04-执行衍生与特殊救济 | 特殊情形 | 执行回转/参与分配/协调/救助 |

## 设计原则

1. **单Skill架构**：所有程序、规则、模板内置于一个Skill，用户无需复制项目结构
2. **自动路由**：AI根据案件材料自动识别状态，自动进入正确程序/阶段
3. **自动询问**：信息缺失时自动进入Ask模式，按优先级收集必要信息
4. **法律数据库强制检索**：引用法条和案例前必须调用专业数据库核实，严禁编造
5. **案由插件化**：特殊案由通过适配规则差异化调整，保持四阶段骨架统一
6. **原始材料保护**：AI不移动、不修改、不删除原始文件，只复制整理
7. **双版本设计**：关键文书区分"内部工作版"和"提交法院版"
8. **AI能力边界**：AI生成的是初稿，须经律师审核后方可提交法院
9. **期限优先规则**：法院指定的期限优先于法定期限
10. **质量检查闭环**：每个阶段产出完成后自动执行质量检查，不通过则修正

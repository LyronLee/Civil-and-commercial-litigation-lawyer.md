# 04 收官与执行：从庭后到闭环

> 核心理念：庭审结束不等于案件结束，但也并不意味着所有工作都要立刻做完。本阶段分四个时点推进：开完庭后先补充代理词；收到判决后根据结果处理上诉/执行；判决生效后办理退费；案件终局后再完成结案归档和复盘。

| **时点二：收到判决书后** | "收到判决了" / "判决结果出来了" / "帮我看看怎么应对" | **工作流 B：裁判应对（上诉/执行二选一）** | `04-阶段产出/02-上诉状.md` 或 `04-阶段产出/02-强制执行申请书.md` |
| **时点三：判决生效后** | "判决生效了" / "帮我写退费申请" / "怎么退诉讼费" | **工作流 C：诉讼费退费** | `04-阶段产出/03-退费申请书.md` |

## 工作流 A：庭后代理词（时点一：开完庭后）

### A.0 确认触发条件

用户明确表达"开完庭了"或类似意思，或主动要求撰写代理词。

> 如用户提到"收到判决了"，应跳转到**工作流 B**，不要执行工作流 A。

### A.1 读取庭审卷宗

1. 读取阶段三产出：
   - `../03-庭审交锋/02-阶段产出/03-庭审纪要.md` — 获取庭审中法官询问、双方回答、举证质证摘要
   - `../03-庭审交锋/02-阶段产出/02-开庭提纲.md` — 获取预判的争议焦点与实际庭审的对比
2. 读取阶段二产出：
   - `../02-诉讼材料及来文管理/06-阶段产出/01-起诉状.md` — 确认诉讼请求和事实基础
   - `../02-诉讼材料及来文管理/06-阶段产出/02-证据目录.md` — 确认证据体系
3. 读取阶段一产出：
   - `../01-战略把脉/03-阶段产出/04-法条汇编.md` — 引用法律依据

### A.2 撰写代理词

**输出文件**：`04-阶段产出/01-代理词.md`

> 代理词是庭审后提交给法院的书面意见，用于系统梳理庭审中已查明的事实和法律，弥补庭审中可能未充分表达的内容，强化法官心证。

**撰写要点**：
1. **首部**：案号、当事人、代理人信息
2. **案件事实综述**：基于庭审纪要，客观陈述庭审中已确认的事实脉络。不引入新的事实主张。
3. **争议焦点回应**：逐一回应庭审中法官归纳的争议焦点，结合庭审中确认的证据进行论证
4. **法律适用分析**：引用法条汇编中的依据，结合案件事实进行法律分析
5. **结论**：明确请求法院支持的诉讼请求

> ⚠️ **关键原则**：代理词应基于庭审中已经查明的事实和双方确认的证据，**避免提出新的事实主张**（除非有正当理由且对方无异议）。代理词的价值在于"系统梳理"和"强化论证"，而非"补充新料"。

### A.3 质量检查

- [ ] 代理词是否基于庭审纪要中的事实，无新的事实主张
- [ ] 是否回应了庭审中法官归纳的全部争议焦点
- [ ] 法律引用是否准确，与法条汇编一致
- [ ] 是否标注"初稿，需律师审核后使用"
- [ ] 是否提示用户：代理词应在判决前提交，逾期可能不被采纳

## 工作流 B：裁判应对（时点二：收到判决书后）

### B.0 确认触发条件

用户明确表达"收到判决了"、"判决结果出来了"或类似意思，或主动将判决书放入`02-法院来文/`。

> 如用户尚未收到判决书（仅开完庭），应跳转到**工作流 A**，不要执行工作流 B。  
> 如用户询问退费、结案或复盘，但未明确收到判决，应先确认判决状态。

### B.1 读取全部卷宗，确认判决结果

1. 读取本阶段 `02-法院来文/` 中的判决书/裁定书。
2. 读取阶段一至三的全部卷宗材料，建立完整案件信息视图。
3. 重点确认：
   - 判决结果（胜诉/败诉/部分胜诉）及具体判项
   - 上诉期限（判决书送达之日起15日，裁定书10日）
   - 履行期限（判决确定的履行期间）
   - 诉讼费承担方式
   - 是否涉及诉讼费退费

### B.2 裁判应对（二选一）

根据判决结果，选择执行以下步骤之一：

#### 选项 A：撰写上诉状（不服一审判决/裁定）

**输出文件**：`04-阶段产出/02-上诉状.md`

**触发条件**：
- 判决结果未达到预期（全部败诉或部分败诉）
- 当事人决定上诉
- 在上诉期限内（判决书送达之日起15日，裁定书10日）

**撰写要点**：
1. **首部**：当事人信息、原审法院、案号
2. **上诉请求**：明确请求二审法院撤销/改判/发回重审的具体判项
3. **上诉理由**：
   - **事实认定错误**：原审法院对哪些事实认定有误，正确事实应是什么
   - **法律适用错误**：原审法院适用了哪些错误的法律条款，应适用什么条款
   - **程序违法**：原审是否存在违反法定程序的情形
   - **证据问题**：关键证据是否被错误采信或排除
4. **尾部**：上诉人签名/盖章、日期

> ⚠️ **上诉期限提醒**：判决书送达之日起**15日内**必须提起上诉，逾期不上诉，一审判决即生效。请务必在上诉期限届满前提交上诉状。

#### 选项 B：撰写强制执行申请书（胜诉后对方不履行）

**输出文件**：`04-阶段产出/02-强制执行申请书.md`

**触发条件**：
- 判决结果达到预期（全部胜诉或部分胜诉）
- 对方在履行期限内未履行义务
- 判决已生效（上诉期满无人上诉，或二审判决已作出）

**撰写要点**：
1. **首部**：申请人、被申请人信息
2. **申请事项**：明确申请强制执行的具体内容（金额、行为、交付财物等）
3. **事实与理由**：
   - 判决结果概述（案号、判项内容）
   - 判决生效时间
   - 履行期限及对方未履行的事实
   - 申请强制执行的依据（《民事诉讼法》第二百四十七条等）
4. **财产线索**（如有）：被申请人的银行账户、房产、车辆等财产信息
5. **尾部**：申请人签名/盖章、日期

> ⚠️ **申请执行期限**：二年，从法律文书规定履行期间的最后一日起计算。不要错过执行时效。

> 退费、结案和复盘不属于本工作流，分别在**工作流 C**（时点三）和**工作流 D**（时点四）中处理。

### B.3 质量检查

生成文件后，逐项核对：

- [ ] 上诉状是否针对原审具体判项，理由具体明确
- [ ] 强制执行申请书是否包含生效判决的关键信息、履行期限、财产线索
- [ ] 全部文件编码为 UTF-8，中文显示正常
- [ ] 裁判应对二选一：上诉状和强制执行申请书不会同时生成

## 工作流 C：诉讼费退费（时点三：判决生效后）

### C.0 确认触发条件

用户明确表达"判决生效了"、"帮我写退费申请"、"怎么退诉讼费"或类似意思。

> 如判决尚未生效（刚收到判决书，对方可能还在上诉期内），应提示用户：退费通常需要判决生效后才能办理，建议先等待判决生效或确认对方不上诉。

### C.1 读取材料

1. 读取本阶段 `02-法院来文/` 中的判决书/裁定书（确认诉讼费承担方式）。
2. 读取阶段二 `../02-诉讼材料及来文管理/05-法院来文/` 中的诉讼费交纳通知书/缴费凭证（确认预交金额）。
3. 确认当事人收款账户信息（需用户补充提供）。

### C.2 生成退费申请书

**输出文件**：`04-阶段产出/03-退费申请书.md`

**触发条件**：
- 判决确定诉讼费由对方承担（或部分承担）
- 原告已预交全部诉讼费

**需要材料**：
- 退费申请书
- 原缴费凭证（诉讼费交纳通知书/收据）
- 判决书/裁定书（证明诉讼费承担方式）
- 收款账户信息（开户名、开户行、账号）
- 申请人身份证明/营业执照

> 各地法院退费流程不同，具体以受理法院要求为准。

### C.3 质量检查

- [ ] 退费申请书是否包含准确的预交金额、判决确定的承担方式、退费金额计算
- [ ] 是否提示用户补充收款账户信息（如尚未提供）
- [ ] 是否注明"具体以受理法院退费要求为准"

## 工作流 D：结案与复盘（时点四：案件终局后）

### D.0 确认触发条件

用户明确表达"案子结了"、"帮我写结案报告"、"复盘一下这个案子"或类似意思。

> **案件终局**指：判决已生效且履行完毕、或执行完毕、或达成调解/和解并履行、或上诉/再审程序结束。如案件尚未终局（如仍在执行中），可先生成结案报告的初稿，复盘总结建议等终局后再完善。

### D.1 读取全部卷宗

读取阶段一至四的全部材料，建立完整案件信息视图：
- 阶段一至三的全部产出
- 本阶段 `02-法院来文/` 中的判决书/裁定书/生效证明
- 本阶段已生成的代理词、上诉状/强制执行申请书、退费申请书

### D.2 撰写结案报告

**输出文件**：`04-阶段产出/04-结案报告.md`

**内容框架**：
1. **案件基本信息**：案号、案由、当事人、承办律师
2. **案件经过**：从接案到结案的完整时间线
3. **裁判结果**：判决/裁定/调解书的主要内容
4. **诉讼费用**：预交金额、承担方式、退费情况
5. **文书归档清单**：本案全部法律文书和证据材料的归档位置
6. **后续事项**：是否上诉、是否申请强制执行、是否已履行完毕
7. **结案状态**：□ 已履行完毕　□ 执行中　□ 上诉中　□ 其他

### D.3 撰写复盘总结

**输出文件**：`04-阶段产出/05-复盘总结.md`

> 复盘总结是阶段四的**核心增值产出**。AI 基于完整卷宗提供客观分析，律师/当事人补充主观经验，形成可复用的方法论。

#### D.3.3 复盘生成的时机

- **最佳时机**：案件彻底终局后（判决生效且履行完毕，或执行完毕）

## 注意事项

1. **分步触发，不要提前**：四个时点各自独立触发。开完庭后只写代理词；收到判决后处理上诉/执行；判决生效后办理退费；案件终局后完成结案和复盘。提前生成没有实际意义。
2. **上诉期限不可错过**：判决书送达之日起**15日内**，裁定书送达之日起**10日内**。逾期不上诉，一审判决即生效。
3. **申请执行期限**：二年，从法律文书规定履行期间的最后一日起计算。不要错过执行时效。
4. **代理词提交时限**：代理词应在判决作出前提交给法院，逾期可能不被采纳。开完庭后应尽快完成。
5. **退费流程因地而异**：务必确认受理法院的具体退费流程和所需材料。
6. **复盘需要主观补充**：AI 的复盘基于文本分析，缺乏庭审现场的直观感受。律师/当事人的手写补充是复盘最有价值的部分。
7. **卷宗长期保存**：结案后，全部卷宗（四个阶段的完整文件夹）应长期保存，保存期限不少于案件诉讼时效期间。
